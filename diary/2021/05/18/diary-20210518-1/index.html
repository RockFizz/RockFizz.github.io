UE4导入FBX骨骼动画

以官方FBX模型小白人UE4_ThirdPerson_FBX.zip为例。

  1. 创建文件夹，骨骼文件和动画文件分开管理。

  2. 导入骨骼文件。
    SK_Mannequin.FBX是骨骼文件。弹出的导入选项可以使用默认。
    导入后自动生成了5个文件资源：

    左1和左2都是材质
    左3是网格模型
    左4是自动生成的物理骨骼，用于制作布娃娃系统(RagDoll System)
    左5是解析出的骨架资源
    UE4中相同骨架的动画是可以直接复用的。

  3. 导入动画文件。
    其余的 FBX文件拖拽到 “Animations”文件夹中。
    导入动画文件时会要求你选择一个骨骼，否则无法导入。在Skeleton选项中，选择刚才生成额骨骼即可。
    导入完毕后会自动解析出动画文件。

动画蓝图和混合空间

动画蓝图：控制所有动画，在什么时候播放，播放什么类型的动画。
混合空间：根据不同的数值，切换不同的动画效果，从一个动画过渡到另一个动画，比如从 站立->走->跑。

本步操作承接上一步导入骨骼和动画之后。
https://blog.csdn.net/lei_7103/article/details/94595523

  1. 新建动画蓝图和混合空间
    Content里右键创建动画蓝图(animation->animation blueprint)，选择父类为AnimInstance类型及目标骨架。命名为Test_Anim。
    Content里右键创建混合空间(animation->BlendSpace ID)，选择与动画蓝图相同的骨骼。命名为Test_BlendSpace。

  2. 设置混合空间
    双击打开混合空间Test_BlendSpace，系统默认窗口布局中，右下角有资源浏览器，选择站立、走、跑动画，拖入到中间区域下方位置，按照顺序拖入，放在不同的分区内，可以通过选中绿色的点左右移动观看动画效果，也可以按住Shift键，左右滑动观看效果
    在左侧资源面板中，AxisSetting面板中设置水平坐标属性：

    Name是指整个动画轴的名字，用来传入值获取到不同值下的动画状态，一般设置名称为Speed
    Minimum Axis Value ：设置轴的最小值
    Maximum Axis Value ：设置轴的最大值（设置400）
    Number of Grid Divisions：区域分割，整体区域分为几个部分，可以用来放置多个动画设置
    Interpolation Time：插值时间，值越大过渡时间越久。为0，两个动画间不存在过渡，从上一个动画直接更改到下一个动画，无动画过渡可能会使动作看起来生硬，设置为0.2。
    Interpolation Type：插值类型（平均，线性），使用默认即可。

  3. 在动画蓝图创建状态机 

双击打开创建的动画蓝图Test_Anim，进入动画蓝图编辑器动画图表界面，在蓝图右键创建 Add New State （创建状态机）节点重命名为State_Machine,将两个白色小人的图标连线  
双击State_Machine节点，进入状态机内，该蓝图内初始值有个Entry节点，状态机从此处开始执行，在此界面可以制作多种状态，通过不同的条件驱动进入不同的状态，蓝图空白处右键创建增加状态节点，命名为Idle_Walk_Run  
增加状态后，双击Idle_Walk_Run节点进入编辑状态界面，进行动画的添加。此状态为站立到走路到跑的状态，所以要加入我们制作的动画空间。在右侧选择资源浏览器，下方是所有动画，也包括我们制作的动画空间，将Test_BlendSpace拖拽到蓝图中  
在混合空间节点中要输入Speed值，该值是需要通过人物角色自身传入的速度来获取动画，所以将该值提升为变量命名为Speed，进行蓝图连接

资源迁移到另一个工程使用

一个项目中的资源需要迁移到另一个项目中去，但是会出现材质丢失图片、材质函数丢失等问题，其实是路径不对。只要将迁移内容放入content文件夹中就不会出现问题。
ContentBrower选中要迁移的资产，右键AssetActions->migrate,迁移到后来建立的Content目录(不要在建子目录了否则不对)下，在桌面或者任意位置新建一个Content文件夹，这样直接替换Content就不会出错。
需要添加到新项目中，直接将项目替换项目content就可以。如果替换失败关闭UE4编辑器之后在替换.迁移的资产可以在另一个UE4工程内使用。

UE4里hub和ui的区别

(HUD)(head-up display抬头显示) 是指在游戏过程中覆盖在屏幕上的状态及信息。HUD的作用是告知玩家 在游戏中的当前状态，也就是，分数、它们的生命值、剩余时间等。 HUD通常是不能交互的，这意味着玩家不能点击HUD的元素，尽管这会变成一个灰色区域， 但在某些类型的游戏中，HUD和用户界面是很难区分的。
(UI)(user interface用户交互界面) 是指菜单和其他的交互性元素。这些元素通常和 HUD 一样覆盖地 描画在屏幕上，但是在某些情况下，用户界面可能会作为游戏世界本身的一部分渲染到 世界中的一个表面上。最明显的用户界面示例是游戏启动时显示的主菜单， 或者玩家暂停游戏时显示的暂停菜单。但是，其他的用户界面可能在游戏过程中显示。这些用户界面 可以用于显示游戏中或者在更复杂的情形(比如RTS或RPG)中两个玩家之间的对话， 它们可以集成到游戏本身中，允许玩家选择武器、弹药、要建立的战队等。

ConstructionScript和EventBeginPlay

ConstructionScript是蓝图构造函数,当你将蓝图类拖放到场景中时，它就被实例化了，这时就会调用一次。而当你拖动物体，旋转物体，缩放物体甚至修改它的参数，对他进行编译时，都会调用这个蓝图构造函数。但是点击播放并不会直接出发构造函数执行。
EventBeginPlay是点击播放游戏开始时进行的初始化事件。
