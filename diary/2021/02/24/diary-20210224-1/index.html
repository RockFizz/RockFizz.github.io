计算机硬件由运算器、控制器、存储器、输入输出设备等组成，系统内核负责驱动硬件、管理活动、分配/管理硬件资源。让各设备各司其职。
系统内核太重要，不能直接被用户操作，需要基于”系统调用接口”开发的程序来满足用户日常工作。shell充当用户与内核(硬件)间的翻译官，用户将命令告诉shell，shell调用内核服务工作。
bash是大多数linux系统默认字符器(shell)。本文linux环境是Deepin。

echo和date

echo命令在终端显示字符串或变量。 

zzz@zzz-PC:~$ zs=123
zzz@zzz-PC:~$ echo $zs
123

//显示本机主机名
zzz@zzz-PC:~$ echo $HOSTNAME
zzz-PC

date命令显示/设置系统时间或日期。键入”+”开头的字符串指定格式。

参数作用
%t跳格(Tab键)
%H小时(00-23)
%I小时(01-12)
%M分钟(00-59)
%S秒(00-60)
%X相当于%H:%M:%S
%Z显示时区
%p显示本地AM或PM
%A星期几(Sunday-Saturday)
%a缩写星期几(Sun-Sat)
%B完整月份(January-Decembeer)
%b缩写月份(Jan-Dec)
%d日(01-31)
%j一年中的第几天(001-366)
%m月份(01-12)
%Y完整年份 
zzz@zzz-PC:~$ date  
2021年 02月 24日 星期三 11:05:08 CST
zzz@zzz-PC:~$ date "+%Y-%m-%d %H:%M:%S"  
2021-02-24 11:09:27  

设置系统时间为2015年9月1日8点半  
zzz@zzz-PC:~$ date -s "20150901 8:30:00"  
查看本地系统时区
zzz@zzz-PC:~$ date "+%Z"
查看星期几  
zzz@zzz-PC:~$ date "+%A"
查看当前是上午还是下午  
zzz@zzz-PC:~$ date "+%p"

wget

使用shell下载网络文件。

参数作用
-b后台下载模式
-O下载到指定目录
-t最大尝试次数
-c断点续传
-p下载页面内所有资源，包括图片、视频等
-r递归下载
-k将下载的HTML页面中的链接转换为相对链接即本地链接
-np不追溯至父级目录 
zzz@zzz-PC:~$ wget https://www.baidu.com/img/flexible/logo/pc/result.png  

//扒站，递归下载整个网站。  
zzz@zzz-PC:~$ wget -r -p -np -k http://***.net/xl**2020/

查看系统状态

获取网卡配置与网络状态  
zzz@zzz-PC:~$ ifconfig  

查看系统内核版本  
zzz@zzz-PC:~$ uname  
zzz@zzz-PC:~$ uname -a  
Linux zzz-PC 5.4.70-amd64-desktop #1 SMP Wed Oct 14 15:24:23 CST 2020 x86_64 GNU/Linux  
内核名称、内核发行版、内核版本、节点名、硬件名称、硬件平台、处理器类型、操作系统等信息。  

查看系统负载情况  
zzz@zzz-PC:~$ uptime 
12:02:58 up 1:27, 3 users, load average: 0.03, 0.07, 0.08  
输出内容分别为系统当前时间、系统已经运行时间、当前在线用户、平均负载值（平均负载分为最近1分钟、5分钟、15分钟的系统负载情况，值越低越好，小于1是正常）
每秒刷新一次获得当前系统负载情况
zzz@zzz-PC:~$ wathc -n 1 uptime

查看当前内存使用量情况,以g为单位

zzz@zzz-PC:~$ cd -

参数作用
-a查看全部文件，包括隐藏
-d仅看目录本身
-h易读的文件容量
-l显示文件详细信息 
zzz@zzz-PC:~$ ls -al

文本编辑

cat命令用来查看纯文本文件(较短)

参数作用
-n显示行号
-b显示行号(不包括空行)
-A显示不可见符号，如tab、空格 
zzz@zzz-PC:~$ cat 文件名

more命令用来查看纯文本文件(较长)

参数作用
-数字预先显示的行数，默认为一页
-d显示提示语句与报错信息 
zzz@zzz-PC:~$ more 文件名

cut命令通过列来提取文本字符

参数作用
-d 分隔符指定分隔符，默认为tab
-f指定显示的列数
-c单位改为字符 
获取当前系统所有用户名称  
zzz@zzz-PC:~$ cut -d: -f1 /etc/passwd  
-d:代表以:作为分隔符，-f1表示只看第一列

获取root用户的默认shell解释器  
zzz@zzz-PC:~$ grep ^root /etc/passwd | cut -d: -f7  
/bin/bash

比较多个文本的差异 

diff A.txt B.txt

文件目录管理

touch命令用于创建空白文件和修改文件时间。
Linux中文件有三种时间:

  • 更改时间(mtime):内容修改时间(不包括权限)
  • 更改权限(ctime):更改权限与属性的时间
  • 读取时间(atime):读取文件内容的时间
参数作用
-a仅修改访问时间atime
-m仅修改更改时间mtime
-d同时修改atime和mtime
-t修改成的时间为[YYMMDDhhmm]

mkdir命令用于创建空白目录

参数作用
-m=MODE默认新建目录权限 如 -m 755
-p连续创建多层目录
-v显示创建的过程 
zzz@zzz-PC:~$ mkdir linuxprobe  
zzz@zzz-PC:~$ mkdir -p a/b/c/d

cp命令用于复制文件或目录。
复制命令有三种情况:

  • 目标文件是一个目录，会将源文件复制到该目录下。
  • 目标文件是一个文件，会将源文件覆盖该文件
  • 目标文件不存在，会将源文件复制并修改为目标文件名称(重命名)
``` zzz@zzz-PC:~$ cp [选项] 源文件 目标文件 ```
参数作用
-p保留原始文件属性
-d如对象为链接文件，则保留该链接文件的属性
-r递归复制(用于目录)
-i若目标文件存在则询问是否覆盖
-a相当于-pdr

mv命令用于移动文件或改名。
rm命令用于删除文件或目录。 

zzz@zzz-PC:~$ mv [选项] 文件名 [目标路径|目标文件]  
zzz@zzz-PC:~$ rm [选项] 文件

rm -rf会删除一切。
rmdir仅仅删除空目录，遇见目录非空会报错。

用户与组管理命令

useradd命令用于创建新的用户。 

useradd [选项] 用户名  

创建名为myusername的用户，并定义家目录路径，UID，指定登录shell解释器(nologin即为不允许登录)  
useradd -d /home/linuxa -u 8888 -s /sbin/nologin myusername
参数作用
-d指定用户的家目录(默认为/home/username)
-D展示默认值
-e账号有效截至日期，格式：YYYY-MM-DD
-g指定一个初始用户组(必须已存在)
-G指定一个或多个扩展用户组
-N不创建与用户同名的用户组
-s指定默认shell解释器
-u指定用户UID

pass命令用于修改用户密码 

修改当前用户密码：  
passwd
修改其他用户密码:
passwd [选项] 用户名
参数作用
-l锁定用户禁止其登陆
-u解除锁定，允许用户登陆
-stdin允许从标准输入修改用户密码，如(echo "NewPassWord" | passwd -stdin Username)
-d使账号无密码
-e强制用户下次登陆时修改密码
-S显示用户的密码状态

userdel命令用于删除用户所有表格。 

zzz@zzz-PC:~$ userdel -d 用户名
参数作用
-f强制删除用户、家目录及相关文件
-r同时删除用户、家目录及相关文件

usermod命令用于修改用户属性。

参数作用
-c填写账号的备注信息
-d -m-d -m连用，可重新指定用户家目录并自动把旧数据转移过去
-e账户到期时间，格式YYYY-MM-DD
-g变更所属用户组
-G变更扩展用户组
-L锁定用户，禁止其登陆系统
-U解锁用户，允许其登陆系统
-s变更默认终端
-u修改用户UID

groupadd命令用于创建群组。 

zzz@zzz-PC:~$ groupadd [选项] 群组名

打包和解压文件

tar命令打包并压缩文件

zzz@zzz-PC:~$ tar -czvf 压缩包名.tar.gz 文件名  

将/etc目录内文件打包并通过gzip格式压缩  
zzz@zzz-PC:~$ tar -czvf etc.tar.gz /etc

tar命令解压并展开压缩包 

zzz@zzz-PC:~$ tar -xzvf 压缩包名.tar.gz  

将etc.tar.gz解压到/root/etc目录  
zzz@zzz-PC:~$ tar -xzvf etc.tar.gz -C /root/etc
参数作用
-c创建压缩文件
-x解开压缩文件
-t查看压缩包内有哪些文件
-z用gzip压缩或解压
-j用bzip2压缩或解压
-v显示压缩或解压的过程
-f目标文件名
-p保留原始的权限与属性
-P使用绝对路径来压缩
-C指定解压到的目录路径

文件内容的查询搜索

grep命令用于对文件内文本进行搜索。 

zzz@zzz-PC:~$ grep [选项] [文件]  
zzz@zzz-PC:~$ grep 关键词 文本文件
参数作用
-b将可执行文件(binary)当作文本文件来搜索
-c仅显示找到的次数
-i忽略大小写
-n显示行号
-v反向选择（列出不含关键词的行） 
找出系统中不允许登录的用户  
zzz@zzz-PC:~$ grep /sbin/nologin /etc/passwd

find命令用于查找文件 

zzz@zzz-PC:~$ find [查找路径] 寻找条件 操作 

搜索在/etc/中所有以host开头的文件  
zzz@zzz-PC:~$ find /etc -name "host*" -print

管道命令符

管道命令符|的作用是将前一个命令的标准输出当做后一个命令的标准输入。格式为”命令A|命令B”: 

搜索找出被限制登陆用户的命令：grep /sbin/nologin /etc/passwd  
统计文本行数的命令： wc -l  
将搜索命令的输出值传递给统计命令，把管道符夹在中间即可。  
zzz@zzz-PC:~$ grep /sbin/nologin /etc/passwd | wc -l  

用翻页的形式查看/etc目录中有哪些文件：  
zzz@zzz-PC:~$ ls -l /etc/ | more  

向linuxprobe用户发送一封邮件  
zzz@zzz-PC:~$ echo "Content" | mail -s "Subject" linuxprobe

管道命令符|并非只能用一次，”命令A|命令B|命令C”也是可以的。

输入输出重定向

  • 标准输入(STDIN,文件描述符0)：默认从键盘输入，为0时表示是从其他文件或命令的输出
  • 标准输出(STDOUT,文件描述符1)：默认输出到屏幕，为1表示是文件
  • 错误输出(STDERR,文件描述符2)：默认输出到屏幕，为2表示是文件
对于输出重定向
符号作用
命令 > 文件将标准输出重定向到一个文件中(清空原有文件内数据)
命令 2> 文件将错误输出重定向到一个文件中(清空原有文件内数据)
命令 >> 文件将标准输出重定向到一个文件中(追加到原有内容的后面)
命令 2>> 文件将错误输出重定向到一个文件中(追加到原有内容的后面)
对于输入重定向
符号作用
命令 < 文件将文件作为命令的标准输入
命令 << 分界符从标准输入中读入，直到遇见分界符才停止
命令 < 文件1 > 文件2 将文件1作为命令的标准输入，将文件2作为命令的标准输出 
zzz@zzz-PC:~$ echo "welcom to BeiJing" > readme.txt

命令行通配符

bash解释器支持多种文本通配符：

通配符含义
*匹配零个或多个字符
?匹配任意单个字符
[0-9]匹配范围内的数字
[abcde]匹配已列出的任意字符
\（反斜杠）转义后面单个字符
''（单引号）转义所有字符(变量无效，不在取值，全当纯字符处理)
""（双引号）变量依然生效
``（反引号）执行命令语句 
查看当前目录下sda开头的所有设备文件：  
zzz@zzz-PC:~$ ls /dev/sda*

查看当前目录下sda后面有一个字符的设备文件：  
zzz@zzz-PC:~$ ls /dev/sda?

查看当前目录下sda后面包含0-9的设备文件：  
zzz@zzz-PC:~$ ls /dev/sda[0-9]

查看当前目录下sda后面是1或3或5的设备文件：  
zzz@zzz-PC:~$ ls /dev/sda[135]

zzz@zzz-PC:~$ PRICE=5
zzz@zzz-PC:~$ echo "Price is $PRICE"
zzz@zzz-PC:~$ echo "Price is \$$PRICE"
zzz@zzz-PC:~$ echo 'Price is \$$PRICE'
zzz@zzz-PC:~$ echo `uname -a`

实用的PATH变量

alias命令用于设置命令的别名，格式为”alias 别名=命令”。
unalias命令用于取消命令的别名，格式为”unalias 别名”。 

zzz@zzz-PC:~$ alias cp="cp -i"  
zzz@zzz-PC:~$ unalias cp

Linux中一切都是文件，命令文件也不例外。当用户键入”ls”命令后发生了什么？

  • 1.如果是以绝对/相对路径输入的命令，直接执行。(如/bin/ls)
  • 2.检查是否为alias别名命令
  • 3.由bash判断是内部命令(属于bash解释器内部)还是外部命令(独立于解释器外的命令)。
  • 4.通过$PATH变量中定义的路径进行命令查找
//查看$PATH变量
zzz@zzz-PC:~$ echo $PATH

//为变量增加新的值  
zzz@zzz-PC:~$ PATH=$PATH:/root/bin  

有经验的运维人员接受一台新的Linux后一定会检查$PATH变量是否有可疑目录

Linux系统中还有许多其他重要的环境变量

变量名称作用
HOME用户的主目录(家目录)
SHELL当前shell是哪个程序

MAIL邮件信箱文件

某个用户设置的环境变量，切换用户后不能被其他用户使用。因为变量存在作用范围。  
export命令用户将局部变量提升为全局变量

zzz@zzz-PC:~$ export v1  

提升为全局变量后，切换用户后别的用户也能识别变量。
