纯Python创建服务器

不安装任何软件实现局域网快速共享文件,只利用python创建一个简易的单线程服务器。

windows的命令行模式下检查python版本。 

python -V

命令行进入想分享的目录，输入以下命令，即可创建以此目录为根目录的http服务器。 

python -m http.server

以下是linux环境下，还未尝试的简化方式。

写个脚本share： 

#!/bin/bash  
deepin-terminal -e "python3 -m http.server" -w $1

把这个脚本放在/usr/bin目录下，即可在任意位置输入“share 路径”来打开服务器。
比如可以实现输入”share ~/Desktop”后以桌面为根目录的分享。

在进一步简化，命令行终端我都不想用，把东西放在右键菜单中实现功能。
这里的目录是：系统盘->/usr/share/deepin/dde-file-manager/oem-menuextensions
我们需要在里边新建一个扩展名为desktop的文件，例如share.desktop，然后在其中输入以下内容 

[Desktop Entry]  
Exec=share %U  
GenericName=share  
GenericName[zh_CN]=分享到局域网  
MimeType=inode/directory;  
Name=share  
Name[zh_CN]=分享到局域网  
Type=Application  
X-Deepin-Vendor=user-custom

Exec那一项表示要运行的命令 %U表示当前的目录，假如说你在桌面点击右键打开，然后这个%U就是~/Desktop。这时候注销后重新登录即可使用。 

deepin-terminal -e "python3 -m http.server" -w $1  
这句命令可能有人不理解，因为python3 -m http.server没有图形界面，  
所以我们需要一个载体来控制他，deepin-terminal是深度终端，-e后边是需要运行的命令，-w是设置当前路径。  
输入deepin-terminal -h可以看到完整的介绍。  
这样做的好处是当你想停止时只需要按Ctrl+C或者直接关掉终端窗口即可。

df -lh  
查看磁盘的使用情况以及文件系统被挂载的位置。

查看机器中所有磁盘的分区情况。

列出当前目录下所有文件及其属性。

显示当前日期和时间。  
cal  
显示日历  

sudo useradd zz  
添加一个用户  
sudo passwd zz  
为新用户zz设置登陆密码(这样仅仅useradd和passwd的新用户没有任何权限)  
su  
切换到root用户  
su zz  
切换到zz用户  

useradd可用来建立用户帐号。帐号建好之后，再用passwd设定帐号的密码．而可用userdel删除帐号。使用useradd指令所建立的帐号，实际上是保存在/etc/passwd文本文件中。  
－c：加上备注文字，备注文字保存在passwd的备注栏中。  
－d：指定用户登入时的起始目录。  
－e：指定账号的有效期限，缺省表示永久有效。  
－f：指定在密码过期后多少天即关闭该账号。  
－g：指定用户所属的起始群组。  
－G：指定用户所属的附加群组。  
－m：自动建立用户的登入目录。  
－M：不要自动建立用户的登入目录。  
－n：取消建立以用户名称为名的群组。  
－r：建立系统账号。  
－s：指定用户登入后所使用的shell。  
－u：指定用户ID号。  

ls -al  
ls(list的意思)显示文件名与属性，-al标识列出所有文件详细权限和属性包括隐藏。  

cp 来源文件 目标文件  
复制文件(复制行为会复制文件的属性和权限)

groups 显示当前用户所在用户组

文件可存取的身份分为3个类别：owner/group/others，3个身份各自有不同的read/write/execute等权限。  

文档类型及权限标识：  
-rwxrwxrwx  
[文件类型][所有者权限][群组权限][其它用户权限]  
第一个字符代表文件类型，可以是普通文件、目录、链接、可存储设备、串口设备  
后续九个字符，每三个一组()，均为rwx组合，r代表可读，w代表可写，x代表可执行，没有权限用减号-占位。  

chgrp  
改变文件所属群组  
要被改变在群组名必须要在/etc/group文件内存在才行  
chown  
改变文件拥有者  
被改向的拥有者必须是已经在系统存在的账号，即在/etc/passwd文件内有记录。  
chmod  
改变文件权限  
文件权限设定有两种方式，  
使用数字修改权限：  
Linxu文件的基本权限就九个。分别是owner/group/others三种身份各自的read/write/execute权限。  
4,2,1的二进制为：  
4:0000 0100  
2:0000 0010  
1:0000 0001  
Linux系统中的文件权限有这样的设计：  用4代表读r，2代表写w，1代表运行x，组合的权限则求和；  修改文件权限只要如下：  
chmod 754 doc1  

754的权限既[rwx][r-x][r--]  
7=4+2+1  
5=4+0+1  
4=4+0+0  
---=0+0+0=0  
--x=0+0+1=1  
-wx=0+2+1=3  
rwx=4+2+1=7  

使用符号修改权限：  
chmod ugoa +-= rwx  
u代表所有者身份，g代表群组身份，o代表其它用户身份，a代表所有身份  
-代表除去，+代表添加，=代表设定，  
r可读w可写x可执行。  

chmod u=rwx,g=rx,o=r doc1  
设定权限  
chmod a-x doc1  
不用管原来权限如何也不会改变，只除掉所有身份的执行权限  
chmod a+x doc1  
不用管原来权限如何也不会改变，只添加所有身份的执行权限  

目录的x可执行权限，意味着能不能通过cd切换进这个目录。

X-window与文本模式切换

文本模式也称为终端接口(Terminal或Console),Linux预设提供六个终端来让用户登录，切换方式为Ctrl+Alt+F1-F6。这六个终端接口被依次命名为tty1-tty6。
Ctrl+Alt+F7返回图形接口X-window。
终端登录时，运行exit进行注销，离开系统。（注销不等于关机，注销只是一个用户离开系统，Linux还有其他的用户在使用并不会关机。）

文件系统与目录树

所谓”挂载”，就是利用一个目录当做进入点，将磁盘分区的数据放在该目录下。也就是说，进入该目录就可以读取对应磁盘分区内的数据。这个动作称为挂载。哪个作为进入点的目录我们称为”挂载点”。
Linxu系统安装时需要规划磁盘分区与目录树的挂载。（决定了以后相应数据放在磁盘的哪个位置）实际上Linux很多发行版已经提供了相当多的默认模式以供选择了。 

最简单的磁盘分区方法：仅分割出根目录(/)和内存置换空间(swap)即可。(一旦磁盘出现小问题，根目录整个损坏，挽救困难)  
麻烦点的磁盘分区方法：根据该主机未来的用途分析需要较大容量的目录，以及读写较为频繁的目录，将这些目录独立出来而不与根目录放在一起。如果读写较频繁的磁盘分区出现问题，至少不会影响到根目录的系统数据，方便挽救。  

默认CentOS环境下，以下的目录属于容量大或读写频繁的目录：  
/  
/usr  
/home  
/var  
Swap

Linux文件类型

linux文件其实可以没有扩展名，能否运行取决于其属性中是否有x。Linux文件名只是让你了解该文件可能的用途，真正执行与否需要权限的规范。
按内容分为以下几类：

  1. [-]
    ls -al显示属性第一个字符为[-].
    纯文本文件(ASCII)，二进制文件(binary),数据格式文件(data)

  2. [d]目录

  3. [l]链接(类似windows下的快捷方式)

  4. 设备与周边，通常集中在/dev目录下，又分为两种：
    block区块设备文件，就是一些储存数据以供系统存取的设备如硬盘软盘等，属性第一个字符为[b]
    character字符设备，就是一些串行接口设备，这些设备特点是一次性读取不能截断输出。属性第一个字符为[c]

  5. 数据接口socket，通常在/var/run目录中，用来在网络上数据承接，启动一个程序监听客户端要求，客户端通过这个socket沟通，属性第一个字符为[s]

根目录/的意义与内容

根目录是整个系统最重要的一个目录，因为不但所有目录都是由根目录衍生出来，同时根目录还跟开机、还原、系统修复等动作相关，若系统出现错误，根目录也必须要包含能够修复文件系统的程序才行，根目录不要放在非常大的分区里，因为分区越大越有可能发生错误，FHS建议根目录/所在分区越小越好且应用程序所安装的软件最好不要与根目录放在同一分区，保持根目录越小越好，不仅效能较佳，根目录所在文件系统也较不容易发生问题。
根目录/下应有以下次目录：

  • /bin
    系统有很多放置执行文件的目录，但/bin比较特殊，因为/bin放置的是在单人维护模式下还能被操作的指令。在/bin底下的指令可以被root与一般账号所使用，主要有cat,chmod,chown,date,mv,mkdir,cp,bash等等指令。
  • /boot
    这个目录主要放置开机会用到的文件，包括Linux核心文件及开机选项、开机配置文件等，如果使用grub这个开机管理程序，则还会存在/boot/grub/这个目录。
  • /dev
    Linux系统内，任何装置与接口设备都以文件形式存在于这个目录中，通过这个目录底下存取某个文件，就等于存取某个装置。这里比较重要的文件有/dev/null,/dev/zero,/dev/tty,/dev/lp*,/dev/hd*,/dev/sd*,等等
  • /etc
    系统主要配置文件几乎放置于此，例如用户账号密码、各种服务启动文件。一般这个目录下的文件可以让一般使用者查阅，但只有root有权力修改。不建议将可执行文件放置于此。 /etc/init.d/ 所有服务的预设启动script放置于此，
  • /home
  • /lib
  • /media
  • /mnt
  • /opt
  • /root
  • /sbin
  • /srv
  • /tmp

使用vi

一般模式：移动光标：hjkl和箭头左下上右一样。输入/后可以查找字符串，按n找下一个
一般模式：dd删除光标当前行，ndd(n为数字)删除光标所在向下n行
一般模式：按yy复制光标当前行，移动光标后按p粘贴
光标移动到第一行，输入3yy，
输入4G，（光标快速移动到第4行）这个时候按下p，就会把第一到三行会粘贴过来

按：进入命令模式，按ESC返回一般模式。:%s/旧内容/新内容/g 可替换内容
