备份博客

在本地在执行hexo_g后，会在博客根目录下生成一个public文件夹，这个文件夹里的文件组合起来就是一个完整的静态博客。接着如果执行hexo_d，就会把这个public文件夹的东西完完整整地拷贝到.deploy_git文件夹里，并把该文件夹里的所有文件全部推送push到远程库，然后部署到网站上。

markdown文章在之前的hexo_g之后，把生成的静态文件拷贝到了.deploy_git文件，但hexo_clean并没能删除·.deploy_git·里的markdown的静态文件，所以每次部署的时候，它还会冒出来。所以同时删掉.deploy_git文件夹下内容即可(其下.git目录的内容除外，不可删除.git目录)。

备份同理，先hexo_clean，然后删除.deploy_git文件夹下除.git目录以外的其它内容，这样备份文件体积也小些。

另外更彻底的瘦身.git目录目录，就是删除.deploy_git文件夹下包括.git的所有目录和文件。然后.deploy_git右键Git Bash Here，执行git init，（把.git删掉，重新init）成为干净全新的库。

hexo自定义页面

方式一: 

$hexo new page "mynewpage"  
会生成**\source\mynewpage\index.md
---  
title: 新页面

---  

使用页面：通过页面主菜单访问  
修改主题_config.yml 文件中的menu  
menu:  
  ***  
  新页面: /mynewpage/ || id-card

方式二 

Hexo默认会对/source/里的所有页面应用主题模板渲染，但有一些前端作品或demo页我们不希望经过渲染，而是能保持完全自定义的样子，那该怎么用Hexo添加自定义的web页面呢?  

使用Hexo提供的跳过渲染配置，适用于整个目录的设置。具体步骤，打开博客根目录_config.yml，找到其中skip_render项，这个项目用来配置/source/中需要跳过渲染的文件或目录  
例如希望跳过/source/projects/里的所有文件渲染，可以配置为：skip_render: "projects/**"

hex文件

hex是16进制的意思，hex文件格式是可以烧写到单片机中、被单片机执行的一种文件格式，可以通过不同的编译器将C程序或者汇编程序编译生成hex。

给单片机烧录程序

（2021-1-29）Keil C51是51系列兼容单片机C语言软件开发系统。

安装软件

  1. 安装Keil软件。选中文件夹中的C51V900安装程序并打开，一路next直到finish完成Keil安装。
  2. 安装USB转串口的驱动程序，选中文件夹中的HL-340安装程序并打开，点击INSTALL自动完成安装。

建立工程

  1. 新建Keil工程，新建文件夹保存工程，打开Atmel选择单片机型号，这里举例比如选择子目录下的AT89C52，点击OK,后续弹出对话框”Copy standard Startup Code to Project Folder and Add file to Project”，点击否。
  2. file->New，新建文件Ctrl+S注意添加后缀.c文件类型。编辑保存文件。
  3. 回到编辑界面，单机Target前的+号，在选项”SourceGroup1”右键菜单选择”AddFilesToGroup ‘SourceGroup1’”。
  4. 弹出对话框选择刚才编辑保存的**.c文件，单击Add,在单击Close。
  5. 单击左侧SourceGroup1前的+号，注意到SourceGroup1文件夹下多了一个子项**.c，此时源代码文件与工程已经关联。
  6. 单击工具栏TargetOptions…，打开Output选项卡并选择Creat_Hex_File，点击ok，这时编译完成可产生供单片机下载的Hex文件。
  7. 单击工具栏Rebuild，可看状态栏显示状态(0错误和0警告)，这时完成了keil工程建立并产生了可下载的hex文件。

COM口检查

  1. 用usb转串口线把电脑usb口和单片机的串口相连。
  2. 设备管理器里确认端口COM*。

Hex文件下载

  1. 打开文件夹”程序下载”压缩包，打开STC_ISP_V481.exe，这里举例弹出的界面选择MCU type为STC89C52RC,单击”打开程序文件”，在你之前保存keil工程的位置找到刚才生成的.hex文件。选择打开。选择COM口为刚才确认的com口。

  2. 单击Download/下载，（之前一定不能先让单片机通电），然后按下单片机的电源开关，让单片机通电，此时开始程序到单片机的下载

英朗应急搭电教程

启动时，如果发动机没有正常工作，车头传来嗒嗒声，同时仪表盘黄色图标亮起，说明电压过低，需要应急搭电来启动发动机。

如果启动发动机发现电瓶电量不足，不要试图多次点火，这样只会加快电瓶剩余电量的消耗，如果电瓶全部消耗光，会使搭线方式启动更困难。

需要施救车或车用搭电宝 

别克全新英朗用的是风帆蓄电池电瓶，带自动启停系统的车辆配备AGM蓄电池。  

电瓶一般使用寿命是3~5年，当然保养得好的还可以更长。  
通过电瓶测量仪或万用表来测量电瓶的电压。  
正常情况下，电瓶的空载电压在13伏左右，满载电压一般不会低于12伏，启动电压不要低于9.5伏。  
电瓶电压偏低，有可能就会出现车辆启动困难或无法启动。  

当然仅使用电压表或万用表的电压挡不加负载来检查蓄电池的存电是否充足，是很难精确的。单单测量电压时，消耗电流极少，故而不会在电池内部产生大的压降，所以显示电压并不低，但若加上相当的负载，如打开大灯（负载电流10～15A）、按喇叭（负载电流6～12A），电瓶便会使灯光暗淡、喇叭沙哑，从而显示出存电不足。  

别克全新英朗电瓶是60安的，带自动启停系统的车辆配备AGM蓄电池。  
别克全新英朗电瓶一般使用年限是3~5年，当然保养得好的还可以更长。在正常驾驶方式下，可使蓄电池保持电量充足，车辆蓄电池无需维护。  
短途驾驶和频繁启动发动机，会使蓄电池放电。  
车辆闲置超过4周，会导致蓄电池放电。  
断开车辆蓄电池负极端子的接线夹。在连接或断开车辆蓄电池之前，请确保点火开关已关闭。  
于带自动启停系统的车辆，务必用另一个AGM蓄电池更换现用AGM蓄电池。AGM蓄电池可用蓄电池上的标签进行识别。


正极符号旁的红色橡胶盖打开后，里面是正极的金属接触片；

正极线连接蓄电池的正极端口，负极线连接负极端口。（此操作应该保证两台车都处于熄火状态。）

接跨接电缆的规则是正极接正极，负极接负极。

进行车对车搭电时，先将正极接好，再接通负极。（切记不要让正负极碰到一起造成短路）

搭电电缆接通后，先启动施救车，随后启动被救车。

拆跨接电缆时要先拆负极，再拆正极。

启动亏电车后，然后取下搭线，大概要让被救车辆着火15分钟左右，保证电池有足够的电量即可

SVN创建分支

创建初始化svn时，将项目包在一个总目录下面。

选中要创建分支的最外层目录(总目录)，右键TortoiseSVN->Branch/tag…
To path 点击三个点，选择新建欲新建的分支名，如”/branches/anyTemperatureFindZero”

以后编辑删除某分支，直接进/branches/下对应分支操作
编辑主分支，直接进/trunk/下操作

SVN分支合并到主干

如果一个主干有多个分支，其中某个分支要合并主干时，先将主干代码合并到此分支后，再将此分支合并到主干。

目的是让当前合并的分支保持和主干一样最新的代码后，再重新将分支新开发的代码合并到主干，这样不会遗漏别的分支合并到主干的代码。

提交错误代码后

假设原始Revsion 4，提交错误代码后变成Revsion 5，
则TortoiseSVN->Show log，选中Revsion 4后右键Revert to this version
修复错误正确后，重新SVN Commit变成Revsion 6，
覆盖掉错误的Revsion 5，既能避免别人更新到错误的提交代码，又能看到曾经的错误提交内容
