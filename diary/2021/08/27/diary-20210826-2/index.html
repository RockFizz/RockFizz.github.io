可以丢弃当前分支的修改

基本名词解释

克隆/新建(clone)：从远程仓库URL加载创建一个与远程仓库一样的本地仓库
提交(commit)：将暂存文件上传到本地仓库（我们在Finder中对本地仓库做修改后一般都得先提交一次，再推送）
推送(push)：将本地仓库同步至远程仓库，一般推送（push）前先拉取（pull）一次，确保一致
拉取(pull)：从远程仓库获取信息并同步至本地仓库，并且自动执行合并（merge）操作，即pull=fetch+merge
获取(fetch)：从远程仓库获取信息并同步至本地仓库
分支(branch)：创建/修改/删除分枝
合并(merge)：将多个同名文件合并为一个文件，该文件包含多个同名文件的所有内容，相同内容抵消
贮藏(git stash)：保存工作现场
丢弃(Discard):丢弃更改,恢复文件改动/重置所有改动,即将已暂存的文件丢回未暂存的文件
标签(tag):给项目增添标签
工作流(Git Flow):团队工作时，每个人创建属于自己的分枝（branch），确定无误后提交到master分支
终端(terminal):可以输入git命令行
每次拉取和推送的时候不用每次输入密码的命令行：git config credential.helper osxkeychain sourcetree
检出(checkout)：切换不同分支
添加（add）：添加文件到缓存区
移除（remove）：移除文件至缓存区
重置(reset)：回到最近添加(add)/提交(commit)状态

使用SourceTree回滚

代码回滚，适用于的场景：
1.提交错代码，想放弃刚刚提交的部分；
2.代码发生冲突，处理比较麻烦，为了代码安全，直接回滚到之前干净的代码。

本地回滚

回滚自己已经提交的代码，但还未推送到远程仓库。 

1.分支->master 点击  
2.在右边的图谱里，选中本地上次拉取的分支  
3.右键，重置当前分支到此次提交，强行合并丢弃所有工作副本改动

远程回滚

回滚远程代码，一定要注意：

  • 想要放弃的代码，是所有开发成员都一致同意的
  • 想要放弃的代码只是自己的，中间没有别人的提交记录，这可以直接回滚
  • 这个操作过程中，提醒其他成员不要推送代码
1. SourceTree开启允许强制推送权限：SourceTree->偏好设置->高级->允许强制推送  
2. 本地回滚（和本地回滚一样，先回滚到想要的位置）  
3. 强制推送代码，切记这个时候不要拉取代码  
4. 完成操作，本地和远程的代码都是你想要回滚的地方

拉取代码前不想提交本地代码

  1. 勾选【未暂存文件】
  2. 点击顶部【贮藏】功能选项
  3. 输入备注信息，贮藏以后就会返回干净的工作区
  4. 贮藏完以后可以直接拉取代码，不会有错误提示
  5. 选择【已贮藏】，选择【应用贮藏】。如果不想要修改直接【删除贮藏】

修改并提交





sourcetree创建本地分支、修改后推送到远程

  1. 远程->直接checkout目标分支到本地，新建目标分支到自己本地子分支

  1. 将当前分支切换到本地子分支，修改代码后，点击推送，将本地子分支推送到远程仓库上

VSCode配合sourcetree提交git

通过sourcetree Checkout分支
通过VSCode 编辑修改提交

通过sourcetree新建仓库，拉取到本地新建分支

通过VScode修改文件，分支内容修改后，点击左侧Source Control，左上角点击加号(Stage Changes)，逐个将修改内容添加进暂存区，
点勾是commit，推送前先pull拉取项目主分支到本地个人子分支（解决冲突后，如果有），点左边圆是push到远程我的分支，发起merge request

通过sourcetree刷新查看分支

当想要拉取到当前分支，而当前分支有未提交(未commit)的修改时，直接拉取会产生冲突。
需要先提交本地修改后，再去拉取。

VSCode使用git

安装VS Code和Git for Windows。

用了git最方便的就是比如在公司写了很多代码后回到家打开vscode只需要点击一下pull就能全部同步过来。是不是很方便。

因为Git是分布式版本控制系统，所以，每个机器都必须自报家门：你的名字和Email地址。 

//git 全局设置(全名和邮箱)  
git config --global user.name "zss"  
git config --global user.email "123456789@qq.com"

在本地创建一个文件夹，从远程库克隆一个本地库： 

cd d:/GitTest   //指定存放的目录
git clone https://git.oschina.net/name/test.git   //你的仓库地址

查看commit日志，回退到某次提交之前 

git log (查看commit日志)  
git reset --hard c1b6ddd9d1dd9f94a3a450a40fe502bc0ff3563(需要回退到的状态的commit标识号)

提交commit本次修改 

git add . (将本次所有修改内容从工作目录添加到暂存区)  
git commit -m "对本次提交的描述"

解决git SSL certificate problem

VScode用git获取项目的时候提示git SSL certificate problem: unable to get local issuer certificate
解决： 

git config --global http.sslVerify false

sourcetree重置当前分支到本次提交

选中状态，右键”Reset master to this commit”

  • 混合合并-保持工作副本并重置索引 === 回退到暂存区
  • 强行合并-保持工作副本并重置索引 === 回退到未暂存区
  • 强行合并-丢弃所有改动过的工作副本 === 直接把提交的文件reset（最好不要用）

第一二项比较安全，不过在你提交过后出现版本及提前又落后的情况下，选择第二项。
如果你在提交之前，有未拉取的代码，这个时候先拉取完之后再commit。否则经过上面的操作，会把别人提交的代码回退过去。

sourcetree合并多次commit

已经在vscode里commit过多次。

  1. 找到所有本地想要合并的提交的下一条
  2. 右键选择重置到此次提交
    使用方式选择”混合合并-保持工作副本但重置索引”
  3. 当前所有本地提交已经回退到未提交状态，一次性提交即可
