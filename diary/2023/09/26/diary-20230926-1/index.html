Android全局静态类提供公共变量

// 用一个单例类存放公共变量
public class RunTime {
    private static volatile RunTime runTime;
    public RunTime(){
        bSatelliteGood=false;
    }
    public static RunTime getInstance(){
        if(runTime==null){
            synchronized (RunTime.class){
                if(runTime==null){
                    runTime=new RunTime();
                }
            }
        }
        return runTime;
    }

    private boolean bSatelliteGood;
    public boolean isbSatelliteGood() {
        return bSatelliteGood;
    }
    public void setbSatelliteGood(boolean bSatelliteGood) {
        this.bSatelliteGood = bSatelliteGood;
    }

}

// 别处使用全局变量时  
    RunTime.getInstance().setbSatelliteGood(true);

    }else{

Android异步任务AsyncTask类

在Android开发中，我们通常将一些耗时的操作使用异步任务的方式进行处理，任务过程中，需要UI进程展示进度。

Android系统为开发人员提供了一个异步任务类（AsyncTask）来实现上面所说的功能，即它会在一个子线程中执行计算任务，同时通过主线程的消息循环来获得更新应用程序界面的机会。

Android UI线程（主线程）有几个特点:  
只能在UI线程操作UI视图，不能在子线程中操作  
不能在UI线程中进行耗时操作，否则会阻塞UI线程，引起ANR、卡顿等问题  

当我们定义一个类来继承 AsyncTask 这个类的时候  
我们需要为其指定3个泛型参数  
AsyncTask<Params, Progress, Result>
Params: 这个泛型指定的是我们传递给异步任务执行时的参数的类型  
Progress:这个泛型指定的是我们的异步任务在执行的时候将执行的进度返回给UI线程的参数的类型。  
Result: 这个泛型指定的异步任务执行完后返回给UI线程的结果的类型  

在定义一个类继承AsyncTask类的时候，必须要指定好这三个泛型的类型，如果都不指定的话，则都将其写成void。

使用AsyncTask的注意事项

1 AsyncTask 的实例必须在 UI Thread 中创建
2 只能在 UI 线程中调用 AsyncTask 的 execute 方法  
3 AsyncTask 被重写的四个方法是系统自动调用的,不应手动调用。  
4 每个 AsyncTask 只能被执行一次，多次执行会引发异常。  
5 AsyncTask的四个方法，只有doInBackground方法是运行在其他线程中,其他三个方法都运行在 UI 线程中，也就说其他三个方法都可以进行 UI 的更新操作。  
6 AsyncTask 默认是串行执行，如果需要并行执行，使用接口 executeOnExecutor 方法

官方例子

private class DownloadFilesTask extends AsyncTask<URL, Integer, Long> {
     protected Long doInBackground(URL... urls) {
         int count = urls.length;
         long totalSize = 0;
         for (int i = 0; i < count; i++) {
             totalSize += Downloader.downloadFile(urls[i]);
             publishProgress((int) ((i / (float) count) * 100));
             // Escape early if cancel() is called
             if (isCancelled()) break;
         }
         return totalSize;
     }
     protected void onProgressUpdate(Integer... progress) {
         setProgressPercent(progress[0]);
     }
     protected void onPostExecute(Long result) {
         showDialog("Downloaded " + result + " bytes");
     }
}

UI主线程中创建对象并调用 execute 执行即可  
new DownloadFilesTask().execute(url1, url2, url3);

子->主线程通信

主线程中实例化Hander对象 

Handler mHandler = new Handler(){
    @Override
    public  void handleMessage(Message msg) {
        super.handleMessage(msg);  
        switch (msg.what) {
            case 0:
                //do something, refresh UI …
            break;  
            default:  
            break; 
        }
    }
};

子线程发送消息给主线程 

new Thread() {
    public void run() {
        Message message = new Message();
        message.obj = "子线程发送的消息";
        mHandler.sendMessage(message);
    };
}.start();

子->子线程通信

在一个子线程中创建一个Handler，回调自然就在此子线程中，然后在另一个子线程中调用这个handler来发送消息。 

new Thread(new Runnable() {
    @Override
    public void run() {
        String msg;
        Looper.prepare();//这里很重要

        childHandler = new Handler() {
            @Override
            public void handleMessage(Message msg) {
                super.handleMessage(msg);
                System.out.println("这是从--->>" + msg.obj+ "过来的消息");  
                switch (msg.what) {
                    case 0:

                    break;  
                    default:  
                    break; 
                }
            }  
        };  
       Looper.loop();//开始轮循 这里很重要
    }
}).start();

创建第二个子线程 

new Thread(new Runnable() {
    @Override
    public void run() {
        Looper loop = Looper.myLooper();//这里很重要
        Message msg = childHandler.obtainMessage();
        msg.obj = " 子线程 ";
        childHandler.sendMessage(msg);
    }
}).start();

主->子线程通信

需要在子线程初始化Looper，并在主线程里创建的Handler引用子线程的Looper
Handler中引用哪个线程的Looper，就在那个线程里处理消息。

public class ThreadHandlerActivity extends Activity{     
    //创建子线程
    class MyThread extends Thread{
        private Looper looper;//取出该子线程的Looper
        public void run() {
             Looper.prepare();//创建该子线程的Looper
             looper = Looper.myLooper();//取出该子线程的Looper
             Looper.loop();//只要调用了该方法才能不断循环取出消息
         }
    }

    private Handler mHandler;//将mHandler指定轮询的Looper

    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.main);
        // thread = new MyThread();
        // thread.start();//开启这个线程

        //实例化一个特殊的线程HandlerThread必须指定一个名字 （MyThread用不上了）
        HandlerThread thread = new HandlerThread("handlerThread");
        thread.start();//开启这个线程

        //下面是主线程发送消息
        mHandler = new Handler(thread.looper){//用18行，thread这里是空
            public void handleMessage(android.os.Message msg) {
                Log.d("当前子线程是----->",Thread.currentThread()+"");
            };
        };
        mHandler.sendEmptyMessage(1);
    }
}

build.gradle报红色波浪线警告因为”Google Play requires that apps target API level 31 or higher.” 

Google Play对targetAPI有最低要求，我们忽略就是了  
把“//noinspection ExpiredTargetSdkVersion”放在targetSdkVersion xx上边，一会儿红波浪线不见了  
//noinspection ExpiredTargetSdkVersion
targetSdkVersion 30

版本号

版本号通常是在应用的build.gradle文件中 

android {
    // ...
    defaultConfig {
        applicationId "com.example.myapp"
        minSdkVersion 16
        targetSdkVersion 30
        versionCode 1 // 这是应用的版本号，可以根据需要更改
        versionName "1.0" // 这是应用的版本名称，可以根据需要更改
        // ...
    }
    // ...
}

在上述示例中，  
versionCode是用于内部版本控制的整数值，  
versionName是显示给用户的版本标识。

指定生成Release版本apk名字

通常是在build.gradle文件中的defaultConfig块中配置的。您可以使用applicationVariants来指定生成APK的名称。以下是一个示例 

android {
    // ...
    defaultConfig {
        applicationId "com.example.myapp"
        minSdkVersion 16
        targetSdkVersion 30
        versionCode 1
        versionName "1.0"
        // ...
    }
    buildTypes {
        release {
            // ...
            applicationVariants.all { variant ->
                variant.outputs.all {
                    outputFileName = "MyApp_Release_v${defaultConfig.versionName}_${variant.buildType.name}.apk"
                }
            }
        }
    }
    // ...
}

工程上传svn前精减体积

如果仓库在window本地，那拉取地址如svn checkout “file:///C:/My SVN Repository”

AndroidStudio工程上传到SVN（Subversion）版本控制时，通常可以删除一些不需要的文件和目录，以减少代码库的大小和混乱。以下是一些常见的可以安全删除的文件和目录：

  1. build目录：
    这是自动生成的构建文件目录，通常包含编译生成的文件。它们可以通过Gradle或其他构建工具重新生成，因此可以从版本控制中排除。

  2. .idea目录：
    这是AndroidStudio的配置目录，包含有关项目的本地配置信息。通常可以排除它，因为其他开发者可以使用他们自己的配置。

  3. local.properties文件：
    这个文件包含本地SDK路径等信息，通常不需要共享。

  4. *.iml文件：
    这些是模块文件，包含有关模块的配置信息。它们可以在构建时自动生成，因此通常不需要版本控制。

  5. .gradle目录：
    这个目录包含Gradle的缓存和其他本地文件。可以考虑将其排除在版本控制之外。

  6. .DS_Store（在Mac上）或Thumbs.db（在Windows上）：
    这些是操作系统生成的文件，可以删除。

  7. build.gradle.kts（Kotlin DSL）或settings.gradle.kts文件：
    如果您不使用Kotlin DSL进行构建和设置，可以删除这些文件。

  8. app/build目录下的generated目录：

Android中的Service

https://blog.csdn.net/liqingtaocsdn/article/details/129028866

相对于Activity是一个有可见界面的、为实现交互而提供的Android应用组件，Service则是一个可以在后台长期运行但又不使用用户界面的Android应用组件。它可以和其他的组件形成一定联系，通过传达信息来联系多个组件共同执行某个操作。举个最简单的例子，你此时正在用手机听歌曲，播放歌曲的进程就是一个后台服务。

在Android中，Service是一种组件，通常用于执行长时间运行的操作，而不需要与用户界面进行直接交互。Services通常用于以下情况： 
1. 后台任务：Service可用于执行后台任务，如下载文件、处理数据、定期执行操作等，而不会干扰用户的应用体验。
2. 音乐播放：音乐播放器通常使用Service来持续播放音频，即使用户切换到其他应用。
3. 网络请求：Service可用于执行网络请求，如向服务器发送数据或接收更新，即使用户关闭了应用。
4. 位置更新：如果您需要在后台持续获取设备的位置信息，可以使用Service来处理位置更新，而不需要前台应用一直运行。
5. 定时提醒：Service可用于创建定时提醒，如闹钟应用，即使用户退出应用也能正常触发提醒。
总之，Service是Android应用的后台工作马，可以让应用执行长时间的任务而不中断用户的前台操作。

自定义BroadcastReceiver子类

package com.example.hdsy.hs;

import android.content.BroadcastReceiver;
import android.content.Context;
import android.content.Intent;
import android.net.wifi.SupplicantState;
import android.net.wifi.WifiManager;
import android.support.annotation.Nullable;
import android.util.Log;

public class WifiScanReceiver extends BroadcastReceiver {
    private static final String TAG = "WifiScanReceiver";
    /**
     * 监听到wifi连接断开事件的回调
     */
    private DisconnectedCB mDisconnectedCB;

    @Override
    public void onReceive(Context context, Intent intent) {
        if (intent == null) {
            return;
        }

        String act = intent.getAction();
        if (WifiManager.SUPPLICANT_STATE_CHANGED_ACTION.equals(act))  {
            handleSupplicantState(intent);
        }
    }

    public void setDisconnectedCB(@Nullable DisconnectedCB disconnectedCB) {
        mDisconnectedCB = disconnectedCB;
    }

    private void handleSupplicantState(Intent intent) {
        // 获取连接状态
        SupplicantState supplicantState = intent.getParcelableExtra(WifiManager.EXTRA_NEW_STATE);

        switch (supplicantState) {
            // 成功
            case COMPLETED:
                // wifiManager.connectionInfo?.ssid?.replace("\"", "")?.let {
                // connectListener.invoke(it, false);
                // }
                Log.i(TAG, "获取连接状态：成功");
                break;

            // 不活跃的
            case INACTIVE:
                Log.i(TAG, "获取连接状态：不活跃的");
                break;
            // 接口禁用
            case INTERFACE_DISABLED: {
                Log.i(TAG, "获取连接状态：接口禁用");
                break;
            }
            case DISCONNECTED: {
                Log.i(TAG, "获取连接状态：断开连接");
                if (mDisconnectedCB != null) {
                    mDisconnectedCB.disconnected();
                }
                break;
            }
            case SCANNING: {
                Log.i(TAG, "获取连接状态：正在扫描");
                break;
            }
            case AUTHENTICATING: {
                Log.i(TAG, "获取连接状态：正在验证");
                break;
            }
            case ASSOCIATING: {
                Log.i(TAG, "获取连接状态：正在关联");
                break;
            }
            case ASSOCIATED: {
                Log.i(TAG, "获取连接状态：已经关联");
                break;
            }
            case FOUR_WAY_HANDSHAKE: {
                Log.i(TAG, "获取连接状态：四次握手");
                break;
            }
            case GROUP_HANDSHAKE: {
                Log.i(TAG, "获取连接状态：组握手");
                break;
            }
            case DORMANT:
                Log.i(TAG, "获取连接状态：休眠");
                break;
            case UNINITIALIZED:
                Log.i(TAG, "获取连接状态：未初始化");
                break;
            case INVALID:
                Log.i(TAG, "获取连接状态：无效的");
                break;
            default:
                Log.i(TAG, "wifi连接结果通知");
                break;
        }
    }

    /**
     * 简单处理WIFI断开事件的回调。
     */
    public interface DisconnectedCB {
        void disconnected();
    }
}
