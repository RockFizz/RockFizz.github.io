GeoServer的MBTiles插件和跨域

win10安装JDK17

双击运行jdk-17_windows-x64_bin.msi

新建系统变量JAVA_HOME值为D:\Java\JDK-17
编辑系统变量Path，添加%JAVA_HOME%\bin
删除jdk17在安装时默认往Path里添加的值C:\Program Files\Common Files\Oracle\Java\javapath

进入命令行java -version 检测jdk17是否安装成功

安装geoserver

geoserver不要安装成window服务，以后手动启动RunManually比较好
默认账号admin/geoserver,端口我使用8089  

第一次试着以手动运行安装后，在开始菜单栏中启动GeoServer时会报错geoServer Error: Unable to access jarfile start.jar，只能到安装目录下自行运行java -jar start.jar

新建一个批处理脚本startgeoserver.bat内容如下
java -jar start.jar

放到GeoServer的安装目录下,双击批处理开启服务

启动服务后，进入web管理页，登录默认管理账号

安装MBTiles插件

前一步可登录服务web管理员确认后，重启电脑。

两个插件：geoserver-2.24-SNAPSHOT-mbtiles-plugin.zip和geoserver-2.24-SNAPSHOT-mbtiles-store-plugin.zip
将这两个插件下载后解压将里面的jar包放入geoserver\WEB-INF\lib路径下
geoserver-2.24-RC-wps-plugin.zip的插件（版本根据自己的来），
同样解压后将jar包放入geoserver\WEB-INF\lib路径下。

重启geoserver或者tomcat，
服务里有WPS
创建数据存储仓库时就有了mbtiles的类型选项，创建MBTiles矢量切片和创建MBTiles栅格切片

根据mbtiles文件发布wms服务

  1. 进入geoserver的web管理页面，新建工作空间，命名，默认工作空间，勾选服务WMS和WPS,点击保存(正常不会有星号)

  2. 添加新的存储仓库，新建数据源选择栅格数据源MBTiles-MBTilesplugin，选择工作空间和数据源名称命名，重点是填写连接参数，点击保存

存储仓库连接参数填写如  
file:data/mytest/WuhanTest.mbtiles   

提前将离线数据文件WuhanTest.mbtiles放到geoserver安装的默认数据文件目录下，本例为
D:\ProgramData\GeoServer\data\mytest\WuhanTest.mbtiles

  1. 新建图层，选择工作空间，命名图层，编辑图层其他属性，发布/再次发布

  2. 点击图层预览，选择刚才发布的图层，点击OpenLayers,预览web服务访问接口 

如http://192.168.37.137:8089/geoserver/mytest/wms?service=WMS&version=1.1.0&request=GetMap&layers=mytest%3Ageotools_coverage&bbox=1.269267429486667E7%2C3536894.172811672%2C1.2758868761361673E7%2C3607827.7350603254&width=716&height=768&srs=EPSG%3A3857&styles=&format=application/openlayers

apk反编译

准备3个工具 

apktool_2.8.1.jar (提取图片文件和布局文件，用于获取资源文件)  
dex2jar-2.0.zip (改后缀解压出classes.dex转化成jar文件，用于获取源码文件)
jd-gui-windows-1.6.6.zip (用于查看jar包内源码)  

1. 获取资源文件  
cmd进入apktool_2.8.1.jar所在目录  
执行java -jar apktool_2.8.1.jar d -f F:\Download\temp\test\xxx.apk -o MMTS
将反编译生成的文件存放到当前目录的MMTS目录内
其中MMTS/res内有drawable、layout、values资源，MMTS/AndroidManifest.xml配置信息  

2. 获取源码文件  
APK后缀名改为.rar或者 .zip，并解压，得到其中的classes.dex文件  
将classes.dex放到dex2jar-2.0.zip解压缩后与d2j-dex2jar.bat同一级目录下  
cmd进入d2j-dex2jar.bat当前目录执行d2j-dex2jar classes.dex将classes.dex反编译成jar包  
得到classes-dex2jar.jar

3. 查看jar包内源码  
jd-gui.exe直接打开classes-dex2jar.jar即可查看源码

加壳、脱壳工具

ASPack能对win32可执行程序压缩加壳。 360加固工具直接对apk文件加固加壳，挺好用

加壳原理

加壳是指将原始的应用程序代码和数据进行加密或混淆处理，并嵌入一个外壳程序中，使得外部用户无法直接访问和修改应用程序的代码和数据  
1. 应用程序代码和数据加密处理  
首先，应用程序代码和数据需要进行加密处理，以防止被非法访问和修改。加密算法可以选择对称加密算法或非对称加密算法，其中对称加密算法加密速度快，但密钥需要传输，容易被攻击者获取；非对称加密算法加密速度慢，但密钥不需要传输，更加安全。  

2. 外壳程序嵌入加密后的应用程序  
其次，将加密后的应用程序代码和数据嵌入一个外壳程序中，形成一个新的可执行文件。这个外壳程序可以是一个简单的壳程序，也可以是一个完整的虚拟机，用于解密和执行应用程序代码和数据。  

3. 动态解密和执行  
最后，当用户启动加壳后的应用程序时，外壳程序会首先解密应用程序代码和数据，并将其加载到内存中。然后，外壳程序会启动一个虚拟机来执行应用程序代码和数据，同时还会进行一些安全检查和防护措施，以保护应用程序的安全性。

安卓app加壳方式

1. 静态加壳  
静态加壳是指在编译应用程序时，通过修改编译器或链接器的参数，将外壳程序嵌入到应用程序中。这种方式可以直接生成一个加壳后的应用程序，但容易被逆向工程和破解。  

2. 动态加壳  
动态加壳是指在运行时，通过加载外壳程序和应用程序代码，来实现加壳的过程。这种方式可以动态地加载和解密应用程序代码和数据，提高了安全性，但也增加了一定的运行负担。  

3. 多层加壳  
多层加壳是指将多个外壳程序嵌入到应用程序中，以增加破解难度。每一层外壳程序都会对应用程序代码和数据进行加密和混淆处理，同时还会添加一些安全检查和防护措施，以防止被破解。  

4. 虚拟机加壳  
虚拟机加壳是指通过嵌入一个虚拟机来执行应用程序代码和数据，从而提高应用程序的安全性。虚拟机可以对应用程序代码和数据进行加密和混淆处理，同时还可以实现一些高级的安全检查和防护措施。

AS中使用ProGuard混淆代码

实践感觉不好用，还是用第三方加固工具比较保险

打开项目

在Android Studio中打开您的Android项目

配置ProGuard文件

在项目的根目录下，找到或创建一个名为proguard-rules.pro的ProGuard配置文件。这个文件包含了指定哪些代码和资源需要被混淆、保护或优化的规则。您可以在其中添加自定义规则，也可以使用默认的规则。

通常，AndroidStudio已经为您的项目创建了一个默认的ProGuard文件。您可以在build.gradle文件中的buildTypes块中找到ProGuard文件的引用： 

buildTypes {
    release {
        minifyEnabled true
        proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
    }
}

以上表示在Release构建中启用ProGuard，并使用proguard-rules.pro文件中的规则

配置ProGuard规则

打开proguard-rules.pro文件，您可以添加自定义的ProGuard规则。这些规则指定哪些类、方法、字段或资源应该被混淆或保护。例如： 

-keep class com.example.myapp.model.** { *; }
-keepclassmembers class com.example.myapp.model.** { *; }

以上规则表示保留com.example.myapp.model包中所有类及其成员不被混淆

运行构建

在Android Studio中，选择Build > Build Bundle(s) / APK(s) > Build APK(s)。这将生成带有ProGuard混淆的APK文件。

查看输出

生成的APK文件将位于app/build/outputs/apk/release/目录中（如果您选择Release构建类型）。您可以使用APK反编译工具或查看生成的APK以确保混淆和优化已经生效。

请注意，ProGuard的配置和规则取决于您的项目和使用的库，因此您可能需要根据项目的需求进行自定义。确保混淆不会破坏应用程序的功能，以及要特别小心处理与反射、依赖注入等相关的代码。最后，定期测试混淆后的应用程序以确保它仍然正常运行。

Android中静态库(.a文件和头文件)的制作和使用

前提确保在AndroidStudio中正确配置NDK路径。
过程包括制作静态库、编写C/C++代码、生成头文件、配置Gradle构建脚本以及在Java代码中使用JNI来调用本地c/c++库内容。

编写C/C++代码

首先，你需要编写C/C++代码，包括你想要的功能或算法。假设你有一个名为mylibrary.c的C文件，其中包含一个简单的函数：
先编写源文件mylibrary.c 

#include "mylibrary.h"
int add(int a, int b) {
    return a + b;
}

生成头文件

为了能够在Java代码中调用本地方法，你需要生成与JNI本地方法相关的头文件。打开命令行，进入项目的根目录，并执行以下命令： 

javah -jni -classpath app/build/intermediates/javac/debug/classes com.example.myapp.MyClass
这将生成一个名为com_example_myapp_MyClass.h的头文件，其中包含本地方法的声明。

javah是一个Java开发工具，用于生成C头文件，以便在Java本地接口（JNI）中使用。 

参数-jni选项告诉javah生成适用于JNI的头文件。  
参数-classpath选项用于指定Java类路径，告知查找类文件的位置。当使用javah生成头文件时，需要提供类路径以确保javah能够找到所需的类和接口  

假设我们有一个Java类Example，它位于com.example包中，并且使用了JNI来调用本地方法  
javah -jni -classpath . com.example.Example  
-classpath .告诉javah在当前目录中查找类文件。com.example.Example是要生成头文件的类的完全限定名。  

执行上述命令后，javah将生成一个名为com_example_Example.h的头文件，其中包含与Example类中定义的本地方法对应的C函数声明,注意声明本地方法  
这些函数声明可以在C或C++代码中使用，以便实现本地方法的功能

编写Android.mk文件

在项目的app/src/main/jni目录中创建一个名为Android.mk的文件，并配置它以编译你的C/C++代码并生成静态库文件（.a文件）。以下是一个示例Android.mk文件的内容 

# 设置模块名称  
LOCAL_PATH := $(call my-dir)
include $(CLEAR_VARS)
LOCAL_MODULE    := mylibrary

# 设置编译的源文件 
LOCAL_SRC_FILES := mylibrary.c

# 设置编译选项  
include $(BUILD_STATIC_LIBRARY)

以下是Android.mk的另一个示例 

# 设置模块名称  
LOCAL_PATH := $(call my-dir)  
include $(CLEAR_VARS)  
LOCAL_MODULE := my_module  

# 设置编译的源文件  
LOCAL_SRC_FILES := $(call all-java-files-under, src)  

# 设置编译的依赖关系  
LOCAL_STATIC_JAVA_LIBRARIES := android-support-v4  

# 设置编译选项  
include $(BUILD_STATIC_JAVA_LIBRARY)

编译本地库

在命令行中进入app/src/main/jni目录，并执行以下命令，编译你的本地库 

ndk-build
//将生成一个libmylibary.a的静态库文件

配置Gradle构建脚本

将静态库引入你的Android应用 

android {
    // ...
    defaultConfig {
        // ...
        externalNativeBuild {
            ndkBuild {
                path "src/main/jni/Android.mk"
            }
        }
    }
    // ...
}

// 在android块之外添加以下配置，指定静态库的位置
android.ndk {
    moduleName "mylibrary"
    abiFilters "armeabi-v7a", "arm64-v8a", "x86", "x86_64"
}

// 在android块之外添加以下配置，指定静态库的位置
android.sources {
    main {
        jni {
            source {
                srcDir 'src/main/jni'
            }
        }
    }
}

在Java代码中加载本地库

在你的Java代码中，使用System.loadLibrary方法加载本地库。假设你的Java类是MyClass.java 

public class MyClass {
    //加载本地库
    static {
        System.loadLibrary("mylibrary");
    }
    //声明本地方法
    public native int add(int a, int b);  
}

在Java代码中调用本地方法

public class MainActivity extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        MyClass myClass = new MyClass();
        int result = myClass.add(5, 7);
        Log.d("JNI", "Result from native code: " + result);
    }
}

1. Download pmd-dist-7.0.0-rc4-bin.zip （https://github.com/pmd/pmd/releases/download/pmd_releases%2F7.0.0-rc4/pmd-dist-7.0.0-rc4-bin.zip）  

2. Extract the zip-archive, e.g. to D:\iTools\pmd-bin-7.0.0-rc4  
3. 将D:\iTools\pmd-bin-7.0.0-rc4\bin添加进系统Path变量。
4. 在pmd安装目录/lib下找到pmd-java-7.0.0-rc4.jar，修改后缀为rar，解压出来，如，为了得到针对java的规则集文件F:\Desktop\pmd-java-7.0.0-rc4\rulesets\java\quickstart.xml  

5. 进入cmd命令行，执行pmd.bat check -d F:\Projects\CBB\new\trunk\development\sourcecode\main\libwaypointplan\src -R F:\Desktop\pmd-java-7.0.0-rc4\rulesets\java\quickstart.xml -f text > result.txt  
生成java的静态分析报告

其中
pmd.bat -d <source_directory> -R <ruleset_file> -f text > pmd_analysis_result.txt
<source_directory>指定代码所在目录  
<ruleset_file>指定规则集文件  
-f text指定分析结果输出格式
> pmd_analysis_result.txt 指定分析结果保存文件

Java自定义异常类来抛出和处理特定的异常情况

Java中，你可以自定义异常类来抛出和处理特定的异常情况。以下是一个简单的示例，演示如何创建和抛出自定义异常：

public class MyCustomException extends Exception {
    public MyCustomException(String message) {
        super(message);
    }
}

public class MyClass {
    public void doSomething() throws MyCustomException {
        // 在某些条件下，抛出自定义异常
        if (someCondition) {
            throw new MyCustomException("发生了自定义异常");
        }
    }

    public static void main(String[] args) {
        MyClass obj = new MyClass();
        try {
            obj.doSomething();
        } catch (MyCustomException e) {
            System.out.println(e.getMessage());
        }
    }
}
