重温nodejs和vue

下载安装nodejs

下载node-v14.18.3-x64.msi，双击安装即可。
中途的"Automatically install necessary tools..."不用勾选。  
node -v  
npm -v  出现版本号即安装成功

创建全局安装目录和缓存日志目录 

在nodejs的安装目录下分别创建node_global文件夹和node_cache文件夹(两个新建目录与安装目内的node_modules同级)  
打开命令提示符，将npm的全局模块目录和缓存目录配置到我们刚才创建的两个文件夹目录下  
npm config set prefix 安装目录\node_global   
npm config set cache 安装目录\node_cache

修改源为淘宝镜像，后续就不需要再安装cnpm了。 

npm config set registry https://registry.npm.taobao.org

配置环境变量 

我的电脑-->系统属性-->高级系统设置  
用户变量 -> Path -> 添加"C:\User\用户名\AppData\Roaming\npm"或者"安装路径\node_global"

环境变量 -> 新建变量NODE_PATH,变量值"安装路径\node_global\node_modules"  
此时node_modules文件夹还未生成，但是等我们把模块安装到全局目录下面时，系统会自动生成该文件夹。 
系统变量 -> Path -> 添加%NODE_PATH%

安装依赖包的区别

进行某项目开发时，npm install 命令安装依赖包时，可以使用-D或-S后缀 

npm install 依赖包 -D
这些依赖项通常是用于项目开发、构建、测试和调试的工具、库或辅助工具。
它们在开发过程中起作用，但在项目部署或发布时通常不会包含在最终的生产环境中
npm install 依赖包 -S  
这些依赖项是项目在运行时所需的核心库或功能模块。  
它们是项目正常运行所必需的，会包含在最终的生产环境中  

在 package.json 文件中，依赖项将被保存在不同的字段中：
开发环境依赖项保存在 devDependencies 字段中。
生产环境依赖项保存在 dependencies 字段中。

离线环境本地PC运行Alpaca和LLaMA

Alpaca是由斯坦福大学开发的小型人工智能语言模型,独特之处在于它的规模和成本效益。只有70亿个参数，Alpaca和OpenAI的text-davinci-003模型一样好。而且你可以在你的本地电脑上运行它，不需要互联网连接。
Alpaca是在LLaMa上进行训练微调的，LLaMa是Meta的大型语言模型，最近在网上泄露了。而为了训练这个语言模型，科学家们使用OpenAI的”text-davinci-003”模型，生成了52K高质量的自我指导数据。有了这个数据集，他们使用HuggingFace的训练框架对LLaMA模型进行了微调，并发布了Alpaca 7B。你也可以使用Meta的LLaMA模型，斯坦福的Alpaca LLM表现得更好，而且速度也相当快。

硬件要求

你可以在任何像样的机器上使用Alpaca7B。我在我的入门级电脑上安装了Alpaca7B，它运行得相当好。为了让你了解情况，我的电脑采用了第十代英特尔i3处理器，配有256GB的固态硬盘和8GB的内存。对于GPU，我使用Nvidia的入门级GeForceGT730GPU，配备2GB的VRAM。
即使没有专用的GPU，你也可以在本地运行Alpaca。但是，响应时间会很慢。除此之外，还有用户甚至能够在RaspberryPi4这样的小型计算机上运行Alpaca。所以你可以推断，Alpaca语言模型也可以很好地在入门级电脑上运行。

设置软件环境以运行Alpaca和LLaMA

Windows,你需要安装Python、Node.js和C++来开始在你的计算机上离线使用大型语言模型 

1. 下载Python3.10（或以下）。向下滚动并点击 “Windows installer (64-bit)” 来下载安装文件  
启动安装文件，启用 “Add Python.exe to PATH” 旁边的复选框。现在，用所有的默认设置安装Python。  
2. 下载安装Node.js 18.0版（或以上）。在安装程序时，保持一切默认。
3. 下载Visutal Studio2022，确保 “Desktop development with C++” 被启用。点击 “Install“，等待它完成安装。
4. 所有东西都安装完毕后，重新启动你的计算机。接下来，打开 “Command Prompt“，运行以下命令，检查Python和Node.js是否安装成功。两者都应该返回版本号。  
python --version  
node --version

在你的电脑上安装Alpaca和LLaMA模型

一旦你设置好了Python和Node.js，就该在你的电脑上安装运行LLM了。
在你继续之前，请确保终端检测到python和node命令。

安装Alpaca7B模型

打开终端（在我的例子中是命令提示符），运行下面的命令来安装Alpaca 7B LLM模型（大约需要4.2GB磁盘空间）。如果你想安装Alpaca 13B模型，用 13B 代替 7B 。这个较大的模型需要8.1GB的空间。  
npx dalai alpaca install 7B  
现在，输入 “y” 并点击回车。这将开始安装Alpaca 7B模型。  
整个过程将需要20至30分钟，这取决于你的互联网连接和模型的大小。  

立即测试Alpaca模型,运行下面的命令，它将立即启动Webserver。  
npx dalai serve  
在你的电脑上使用网络浏览器，打开以下地址。这将使你进入网络用户界面，在那里你可以在本地和没有互联网的情况下测试Alpaca模型  
http://localhost:3000  
现在你可以在没有互联网连接的情况下在你的电脑上开始使用这个类似ChatGPT的语言模型。将 “PROMPT” 替换成你的查询，然后点击 “Go”。  

如果你想删除下载的模型以释放磁盘空间，请打开你的用户配置文件目录。这里，”dalai” 文件夹有所有的文件，包括模型。删除 “dalai” 文件夹将立即释放空间。

你可以选择同时安装LLaMA模型，记住，LLaMA的体积要大得多。它的7B模型需要多达31GB的空间。要安装它，请运行下面的命令。你可以用 13B, 30B, 和 65B替换 7B。最大的模型需要多达432GB的空间。 

npx dalai llama install 7B  
npx dalai llama install 13B  
npx dalai llama install 30B  
npx dalai llama install 65B  

安装完毕后，同样通过npx dalai serve启动Webserver  
http://localhost:3000右上角的"model"下拉菜单中选择"Alpaca 7B"或"LLaMA 7B"或其它你下载安装的模型。

cmake的安装和使用

安装 

sudo apt-get install cmake  
// 查看版本  
cmake --version

使用 

1. 首选新建工程目录helloworld  
mkdir helloworld  
2. 新建cmake需要的工作目录  
cd helloworld
mkdir bin
mkdir lib
mkdir src
mkdir include
mkdir build
touch CMakeLists.txt  
其中：  
helloworld/bin 保存可执行文件  
helloworld/lib 存放生成的链接库  
helloworld/src 存放源文件  
helloworld/include 存放头文件  
helloworld/build 存放编译过程生成的文件  
3. 新建源文件，新建头文件  
touch src/main.cpp  
touch src/helloworld.cpp  
touch include/helloworld.h  
编辑内容如下：  
//  main.cpp
#include <helloworld.h>
int main()
{
    helloworld obt;
    obt.outputWord();
    return 0;
}

//  helloworld.cpp
#include "helloworld.h"
void helloworld::outputWord()
{
    std::cout << "hello world!" << std::endl;
}

//  helloworld.h
#ifndef HELLOWORLD_H_
#define HELLOWORLD_H_
#include <iostream>
class helloworld
{
public:
    void outputWord();
};
#endif

4. 编写CMakeLists.txt文件：  
cmake_minimum_required(VERSION 2.8)        //cmake最低版本
project(helloworld)        //工程名称
SET(CMAKE_BUILD_TYPE Release)        //设置编译方式debug或release
set(EXECUTABLE_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/bin)        //设置可执行文件保存路径
set(LIBRARY_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/lib)        //设置链接库保存路径  
include_directories(        //设置头文件目录，使系统可以找到对应的头文件
${PROJECT_SOURCE_DIR}/include
)
add_executable(helloworld src/helloworld.cpp src/main.cpp)        //选择需要编译的源文件，凡是要编译的源文件都需要列举出来

假如需要用到/usr/lib下的静态库libmy.a，则需要添加以下两行  
link_directories(/usr/lib) //静态库所在位置  
target_link_libraries(helloworld my)

5. 编译程序  
cd build  
cmake ..
make  
6. 运行程序  
./../bin/helloworld

Ubuntu下boost的hellowrold

“boost/timer.hpp”不适用于高精度的时间测量任务，它的精度依赖操作系统或编译器，难以做到跨平台，每秒的clock数则由宏CLOCKS_PER_SEC定义，在不同的操作系统中其值不同，在win32下，其值是1000，精度为毫秒，而在linux下为1000000，精度为微秒；

//.Pro文件:  
TEMPLATE = app
CONFIG += console c++11
CONFIG -= app_bundle
CONFIG -= qt
LIBS += -lboost_thread -lboost_system
SOURCES += \
        main.cpp

// main.cpp:  
#include <iostream>
#include <boost/timer.hpp>
#include <boost/thread/thread.hpp>
using namespace std;

void runSomeThing(){
    boost::timer t;
    t.restart();
    cout << "max timespan: "<<t.elapsed_max()/3600<<" h " << endl;
    cout << "min timespan: "<<t.elapsed_min()<<" s "<< endl;
    cout << "now time elapsed:" <<t.elapsed()<<" s "<< endl;
    cout << "start record!" << endl;
    t.restart();
    for(auto i=0;i<1000000;i++){
        auto z=i;
        // delay 1 second = 1000 millisecond
        // sleep(1);
        // delay 500 millisecond
        // boost::this_thread::sleep_for(boost::chrono::milliseconds(500));
        cout << "now recording: " <<i<< endl;
    }
    cout << "1 million cycles consumed time:" <<t.elapsed()<<" s "<< endl;
}
int main()
{
    boost::thread thrd(&runSomeThing);
    thrd.join();
    cout << "Hello World! complete." << endl;
    return 0;
}

运行程序

可能会报error while loading shared libraries: libXXXXXXX.so.的错误无法运行

默认情况下，编译器只会使用/lib和/usr/lib这两个目录下的库文件，通常通过源码包进行安装时，如果不指定–prefix，会将库安装在/usr/local/lib目录下；当运行程序需要链接动态库时，提示找不到相关的.so库，会报错。也就是说，/usr/local/lib目录不在系统默认的库搜索目录中，需要将目录加进去。 

sudo vim /etc/ld.so.conf  
在"include ld.so.conf.d/*.conf"下方添加"/usr/local/lib"  
保存退出后  
sudo /sbin/ldconfig -v  
sudo ldconfig

#include <iostream>
#include <json/json.h>
using namespace std;

#include <fstream>

int main()
{
    cout << "read json from file:" << endl;
    ifstream ifs("alice.json");
    Json::Reader read_ali;
    Json::Value obj_ali;
    if(read_ali.parse(ifs,obj_ali)){
        cout << "Book: " << obj_ali["book"].asString() << endl;
        cout << "Year: " << obj_ali["year"].asInt() << endl;
        const Json::Value &characters=obj_ali["characters"];
        for(int i=0;i<characters.size();i++){
            cout << "    name:" << characters[i]["name"].asString();
            cout << " chapter: " << characters[i]["chapter"].asInt();
            cout << endl;
        }
    }else{
        cout << "parse error!!" << endl;
    }

    cout << "read json from string:" << endl;
    string jsonString="{\"name\":\"aaa\",\"age\":18}";
    cout << jsonString <<endl;
    Json::Reader reader;
    Json::Value value;
    if(reader.parse(jsonString,value)){
        string name=value["name"].asString();
        int age=value["age"].asInt();
        cout<< name<<" : "<<age<<endl;
    }else{
         cout << "parse error!" << endl;
    }

    cout << "write json to file" << endl;
    Json::Value root;
    root["name"]=Json::Value("shuiyixin");
    root["age"]=Json::Value(21);
    root["sex"]=Json::Value("man");

    Json::Value friends;
    friends["friend_name"]=Json::Value("ZhaoWuxian");
    friends["friend_age"]=Json::Value(21);
    friends["friend_sex"]=Json::Value("man");
    root["friends"]=Json::Value(friends);

    root["hobby"].append("sing");
    root["hobby"].append("run");
    root["hobby"].append("Tai Chi");

    cout<< "FastWriter:"<< endl;
    Json::FastWriter fw;
    cout << fw.write(root)<< endl <<endl;

    cout<< "StyleWriter:"<< endl;
    Json::StyledWriter sw;
    cout << sw.write(root)<< endl <<endl;

    string jsonstr=sw.write(root);
    cout << jsonstr << endl;

    ofstream os;
    os.open("demo.json");
    if(!os.is_open()){
        cout<< "error: can not find or create the file" << endl;
    }else{
        os << sw.write(root);
        os.close();
    }
    return 0;
}

    tinyxml2::XMLDocument doc;

        return 0;

    tinyxml2::XMLDeclaration* declaration=doc.NewDeclaration();
    doc.InsertFirstChild(declaration);

}
