基于docker使用nginx容器

挂载宿主机文件夹的方式使用nginx容器

获得Nginx容器里原始配置文件 

// 拉取Nginx镜像
docker pull nginx  

// 宿主机新建目录d:\data\nginx\conf

// 生成原始容器
docker run --name nginx1 -p 9001:80 -d nginx  

// 将容器内nginx的配置文件复制到外部宿主机目录内  
docker cp nginx1:/etc/nginx/nginx.conf d:\data\nginx\conf\nginx.conf  
docker cp nginx1:/etc/nginx/conf.d d:\data\nginx\conf\  
// 将容器内nginx的html目录复制到外部宿主机  
docker cp nginx1:/usr/share/nginx/html d:\data\nginx\  

// 删除指定的正在运行的容器  
docker rm -f nginx1  

// 重新启动另一个容器  
docker run -p 9002:80 --name nginx -v /d/data/nginx/conf/nginx.conf:/etc/nginx/nginx.conf -v /d/data/nginx/conf/conf.d:/etc/nginx/conf.d -v /d/data/nginx/log:/var/log/nginx -v /d/data/nginx/html:/usr/share/nginx/html -d nginx:latest

为方便更新，创建容器时采用挂载宿主机文件夹的方式。从脚本可知，外置了nginx的配置文件，日志。
新容器nginx的配置使用被映射到宿主机目录下的d:/data/nginx/conf/nginx.conf和d:/data/nginx/conf/conf.d/default.conf两个文件  
新容器nginx提供的html内容被映射到宿主机目录下的d:/data/nginx/html/  
新容器nginx的运行log可以在宿主机d:/data/nginx/log/目录下查看  
修改宿主机相关目录下的配置文件和html内容，即可使用容器提供的nginx功能  

采用docker部署nginx，实在方便，启动、停止，都很爽。如果直接跑在操作系统下，想停止、重启就麻烦了，需要杀掉相关进程。  
不过，要结合挂载方式才能体现该优越性。否则配置文件也跟着删掉。  

// 修改配置后重启容器 docker restart <容器ID或名称>
docker restart nginx

nginx的反向代理

注意看上面配置文件的这部分，通过匹配URL对用户的访问进行服务端提供内容的重定向，其实就是nginx的反向代理功能 

    location ~ /abc/ { 
       # 要代理去哪个服务器的ip和端口
       # 匹配URL /abc/  
       # 就是原来本机的/ 是http://192.168.37.131:9002/

        proxy_redirect default;

nginx的负载平衡

Nginx除了可以直接作为web服务器使用外，更多的情况是通过反向代理将请求转发给上游服务器。配置上游服务器可以使用upstream进行设置，通过upstream可以实现服务的负载均衡规则指定多台服务器。

nginx作为负载均衡服务器，用户请求先到达nginx，再由nginx根据负载配置将请求转发至后面相应的其他应用服务器上 

worker_processes  1;

events {
    worker_connections  1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;

    sendfile        on;

    keepalive_timeout  66;
    upstream myapp1{ #注意这里，默认情况下nginx按加权轮转的方式将该请求分发到各服务器
        server 192.168.37.201:9090 down; #当前设备不参与负载
        server 192.168.37.202:9091 weight=10; #负载权重为10，默认为1  
        server 192.168.37.202:9093 weight=3; #负载权重为3，默认为1
        server 192.168.37.203:9091 max_fails=3 fail_timeout=30s; #30秒内nginx与服务器通信尝试失败次数超过3次，该服务器被认为不可用
        server 192.168.37.204:9091 backup; #其它非backup服务器down或忙时才请求这台服务器
    }
    server {
        listen       80;
        server_name  localhost;
        location / {
            proxy_pass http://myapp1; #注意这里
            index  index.html index.htm;
        }

    }
}
