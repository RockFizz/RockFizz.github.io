编译源代码

VS2015+QT5.8.0+VTK8.2.0

下载源代码

解压VTK8.2.0.zip，组织目录结构
F:/VTK/VTK_src放下载的源码
F:/VTK/VTK_bin放编译出来的项目文件
F:/VTK/VTK_res是中转文件夹
F:/VTK/VTK_debug放debug版本的内容
F:/VTK/VTK_release放release版本的内容

添加系统变量

Qt5_DIR F:\Qt\Qt5.8.0\5.8\msvc2015_64\lib\cmake\Qt5
Qt5Core_DIR F:\Qt\Qt5.8.0\5.8\msvc2015_64\lib\cmake\Qt5Core
Qt5Gui_DIR F:\Qt\Qt5.8.0\5.8\msvc2015_64\lib\cmake\Qt5Gui
Qt5UiPlugin_DIR F:\Qt\Qt5.8.0\5.8\msvc2015_64\lib\cmake\Qt5UiPlugin
Qt5Widgets_DIR F:\Qt\Qt5.8.0\5.8\msvc2015_64\lib\cmake\Qt5Widgets
将F:\Qt\Qt5.8.0\5.8\msvc2015_64\bin添加进系统的path变量里置顶 // 不是编译源码必须

CMAKE

打开F:\cmake-3.21.4-windows-x86_64\bin\cmake-gui.exe
where is the source code: F:/VTK/VTK_src
where is build the binaries: F:/VTK/VTK_bin
第一次点configure弹窗中Visual Studio 15 2017的Optional platforms for use填写x64
然后
CMAKE_INSTALL_PREFIX 修改为 F:/VTK/VTK_bin/install
VTK_Group_Qt 打钩选中
VTK_QT_VERSION是5
再次点击Configure，看到Configuring done
然后点击Generate，看到Generating done

VS2015编译工程

进入 F:/VTK/VTK_bin，用vs2015打开VTK.sln解决方案
将工程切换到release+x64
选中ALL_BUILD编译，等待一段时间后，在build-bin\lib目录内得到所有内容相应的dll和lib
选中INSTALL编译，等待一段时间后，在（F:/VTK/VTK_bin/install）build-bin\include\lib目录内得到dll\lib.h文件

至此完成vtk源码编译。
之后是qt使用QVTKWidget的内容

把F:\VTK\VTK_bin\bin\Release\QVTKWidgetPlugin.dll
F:\VTK\VTK_bin\lib\Release\QVTKWidgetPlugin.exp
F:\VTK\VTK_bin\lib\Release\QVTKWidgetPlugin.lib
三个文件都放到F:\Qt\Qt5.8.0\5.8\msvc2015_64\plugins\designer目录下
（Qt Designer有x86和x64两种版本，而Qt Creator只有x86一种版本，本文的各种环境均以x64为主，目前的主流也是x64，故在Qt Creator中，添加QVTKWidget就会报错）
解决方案：搭建好Qt+vs2015的环境，将代码放到vs中进行编译，在vs2015菜单Qt VS Tools/lanuch Qt designer下，打开ui设计界面，这里可以看到之前添加的QVTKWidget显示控件，在这进行拖动添加，在vs上进行编译，就可以使用该控件。  
含有QVTKWidget组件(我编的是x64版本)的qt工程，必须通过vs编译才行（所谓VS2015+QT+VTK）

配置vs2015编译使用VTK的QT工程

增加包含目录

项目->属性->VC++目录->包含目录 添加F:\VTK\VTK_bin\install\include\vtk-8.2
为了找到(.h)头文件

增加库目录

项目->属性->VC++目录->库目录 添加F:\VTK\VTK_bin\lib\Release
为了找到(.lib)头文件

因为该目录下有QVTKWidget.lib,而install下没有

增加依赖附加项

在F:\VTK\VTK_bin\install\lib下创建bat脚本
将生成的***.lib信息集体添加进
项目->属性->链接器->输入->附加依赖项

@echo off
DIR "F:\VTK\VTK_bin\install\lib" /B > libname.txt
pause

main函数修改

注意新添加的三行 

#include "mainwindow.h"
#include <QApplication>

#include <vtkAutoInit.h>             //new
VTK_MODULE_INIT(vtkRenderingOpenGL2)     //new
VTK_MODULE_INIT(vtkInteractionStyle)         //new

int main(int argc, char *argv[])
{
    QApplication a(argc, argv);
    MainWindow w;
    w.show();
    return a.exec();
}

打包运行

将F:\VTK\VTK_bin\install\bin下所有dll复制到打包输出目录下  
通过windeployqt获取qt的相关依赖项

QVTKWidget显示JPG图像

#include "mainwindow.h"
#include "ui_mainwindow.h"
#include <QVTKWidget.h>
#include <vtkImageViewer2.h>
#include <vtkJPEGReader.h>
#include <vtkPNGReader.h>
#include <vtkRenderWindow.h>
#include <vtkRenderer.h>
#include <vtkImageActor.h>
#include <vtkSmartPointer.h>
MainWindow::MainWindow(QWidget *parent) :
    QMainWindow(parent),
    ui(new Ui::MainWindow)
{
    ui->setupUi(this);

    vtkSmartPointer<vtkImageViewer2> m_ImageViewer = vtkSmartPointer<vtkImageViewer2>::New();
    vtkSmartPointer<vtkRenderer> m_Renderder=vtkSmartPointer<vtkRenderer>::New(); //新建渲染器
    ui->qvtkWidget1->GetRenderWindow()->AddRenderer(m_Renderder); //设置qvtkWidget的渲染器
    vtkSmartPointer<vtkJPEGReader> reader = vtkSmartPointer<vtkJPEGReader>::New(); // 用vtkJPEGReader读取JPG图像
    reader->SetFileName("F:\\Desktop\\123.jpg"); 
    reader->Update(); // 不调用update时，调用GetOutput()函数没有数据
    m_ImageViewer->SetInputData(reader->GetOutput()); // 将reader的输出作为m_ImageViewer的输入，并设置m_ImageViewer与渲染器m_Renderer的关联
    m_ImageViewer->UpdateDisplayExtent();
    m_ImageViewer->SetRenderWindow(ui->qvtkWidget1->GetRenderWindow());//设置为同一个渲染窗口
    m_ImageViewer->SetRenderer(m_Renderder);
    m_ImageViewer->SetupInteractor(ui->qvtkWidget1->GetRenderWindow()->GetInteractor());//交互方式
    m_ImageViewer->SetSliceOrientationToXY();//默认为这个方向
    m_ImageViewer->GetImageActor()->InterpolateOff();
    m_Renderder->ResetCamera();
    m_Renderder->DrawOn();
    ui->qvtkWidget1->GetRenderWindow()->Render(); //自动绘制出图像
}

MainWindow::~MainWindow()
{
    delete ui;
}
