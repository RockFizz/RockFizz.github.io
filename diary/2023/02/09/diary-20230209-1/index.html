C++11中，标准委员会赋予了auto全新的含义即：auto不再是一个存储类型指示符，而是作为一个新的类型 指示符来指示编译器，auto声明的变量必须由编译器在编译时期推导而得。
使用auto定义变量时必须对其进行初始化，在编译阶段编译器需要根据初始化表达式来推导auto的实际类 。因此auto并非是一种“类型”的声明，而是一个类型声明时的“占位符”，编译器在编译期会将auto替换为变量实际的类型。

用auto声明指针类型时，用auto和auto*没有任何区别
用auto声明引用类型时则必须加&
auto不能作为函数的参数
auto不能直接用来声明数组

for( auto a:b)与for( auto &a:b)的区别如下:

for(auto a:b)中b为一个容器，效果是利用a遍历并获得b容器中的每一个值，但是a无法影响到b容器中的元素。
for(auto &a:b)中加了引用符号，可以对容器中的内容进行赋值，即可通过对a赋值来做到容器b的内容填充。 

#include<iostream>
using namespace std;
int main()
{
     string s("hello world");
     for(auto c:s)
     c='t' ;
     printf("string = %s \n",s.c_str());//结果为hello world
     for(auto &c:s)
     c='t' ;
     cout<< s.c_str() << endl;  // 结果为ttttttttttt
     return 0;
}

变量名实质是一段连续存储空间的别名，通过变量来申请并命名内存空间，通过变量来使用内存空间。
变量名，本身是一段内存的引用。
引用的语法：Type& name = var;
引用声明时必须初始化，一经声明不可变更。（除非引用作为函数参数时不用进行初始化）
可多次引用，多次引用的结果是一个变量具有多个别名。

&符号之前有数据类型时是引用，其它情况皆是取地址 

#include<iostream>

可以用引用解决的问题，尽量避免使用指针。

引用作为函数返回值时
如果返回的是栈变量，会报错，返回值不能作为其他引用的赋值，不能当右值，不能当左值
如果返回的是全局变量或静态变量，返回值能作为其它引用的赋值，即可当右值用，也可当左值用 

void swap(int a,int b); // 无法实现两个数据的交换  
void swap(int *a,int *b); // 开辟两个指针空间实现交换  
void swap(int &a,int &b){
    int tmp;
    tmp=a;
    a=b;
    b=tmp;
}
int main()
{
    int a=3,b=5;
    printf("a: %d,b: %d\n",a,b);
    swap(a,b);
    printf("a: %d,b: %d\n",a,b);
    return 0;
}

#include<iostream>
struct Teacher{
    char name[64];
    int age;
};
int getTeacher(Teacher **p){
    // **p 指针的指针 作为参数
    Teacher *tmp=NULL;
    if(p==NULL){
        return -1;
    }
    tmp=(Teacher*)malloc(sizeof(Teacher));
    if(tmp==NULL){
        return -2;
    }
    tmp->age=33;
    *p=tmp;
    return 0;
}
int getTeacher2(Teacher *&myp){
    // 指针的引用 作为参数
    myp=(Teacher*)malloc(sizeof(Teacher));
    if(myp==NULL){
        return -1;
    }
    myp->age=36;
    return 0;
}
void FreeTeacher(Teacher *p){
    if(p==NULL){
        return;
    }
    free(p);
}

int main()
{
//    指针的指针(指针的地址)作为参数
    Teacher *pT1=NULL;
    getTeacher(&pT1);
    printf("age: %d \n",pT1->age);
    FreeTeacher(pT1);
//    指针的引用作为参数
    getTeacher2(pT1);
    printf("age: %d \n",pT1->age);
    FreeTeacher(pT1);
    return 0;
}

第二章类和对象

struct和class关键字的区别：
用struct定义类时，所有成员的默认属性为public
用class定义类时，所有成员的默认属性为private

封装有两层含义（把属性和方法进行封装，对属性和方法进行访问控制）
public修饰的成员变量和成员函数可以在类内部和类的外部被访问。
private修饰的成员变量和成员函数只能在类内部被访问。

当类成员中含有一个const对象或者含有一个引用时，必须要通过成员初始化列表进行初始化，因为这两种对象要在声明后立马初始化，而在构造函数中对其进行赋值是不被允许的。初始化列表中的初始化顺序，与声明顺序有关。 

#include<iostream>
using namespace std;
class ABCD{
public:
    ABCD(int a, int b,int c){
        _a=a;
        _b=b;
        _c=c;
        printf("ABCD() construct,a:%d,b:%d,c:%d \n",_a,_b,_c);
    }
    ~ABCD(){
        printf("~ABCD() construct,a:%d,b:%d,c:%d \n",_a,_b,_c);
    }
    int getA(){
        return _a;
    }
private:
    int _a;
    int _b;
    int _c;
};

class MyE{
public:
    MyE():abcd1(1,2,3),abcd2(4,5,6),m(100){
        cout<<"MyE()"<<endl;
    }
    MyE(const MyE &obj):abcd1(7,8,9),abcd2(10,11,12),m(200){
        cout<<"MyE(const MyE &obj)"<<endl;
    }
    ~MyE(){
        cout<<"~MyE()"<<endl;
    }
public:
    ABCD abcd1;
    ABCD abcd2;
    const int m;
};

int doThing(MyE mye1){
    printf("doThing() mye1.abcd1.a: %d \n",mye1.abcd1.getA());
    return 0;
}
int run(){
    MyE mye;
    doThing(mye);
    return 0;
}

{
    run();
    return 0;

使用new和delete运算符动态分配和撤销内存空间 

new 动态分配堆内存运算符  
使用形式：  
指针变量=new 类型(常量); //常量是初始化值，可缺省  
指针变量=new 类型[表达式];  
作用：从堆内存分配一块"类型"大小的存储空间，返回首地址  

delete 释放已分配的内存空间运算符  
使用形式:  
delete 指针变量;  
delete[] z指针变量；  
其中，指针变量必须是一个new返回的指针。  

malloc不会调用类的构造函数，而new会调用类的构造函数  
Free不会调用类的析构函数，而delete会调用类的析构函数

静态成员属于整个类而不是某个对象，静态成员变量只存一份供该类所有对象共用。
类的静态成员属于类，也属于对象，但最终归属于类。

static成员变量实现了同类对象间信息共享
static成员变量在外存储，求类大小时并不包含在内
static成员变量只能在类外初始化
静态成员函数只能访问静态数据成员 

// 声明  
static 数据类型 成员变量; //在类的内部

// 初始化  
数据类型 类名::静态数据成员 = 初值; //在类的外部

// 调用  
类名::静态数据成员  
类对象.静态数据成员

C++实现日志记录

支持日志级别。
支持日志转储至文件。
使用方便，直接使用LOG(XXX)<<接口即可。

日志功能包含两个文件：Logger.h（头文件）和Logger.cpp（实现文件），直接把这两个文件放到工程里后初始化即可使用

logger.h文件 

/**
 * logger.h
 * @brief 简易日志模块
 */
#ifndef __LOGGER_H__
#define __LOGGER_H__

#include <iostream>
#include <iomanip>
#include <fstream>
#include <string>
#include <cstdlib>
#include <stdint.h>

typedef enum log_rank{
    INFO = 0,
    WARNING,
    ERROR,
    FATAL,
} log_rank_t;

void initLogger(const std::string &info_filename);

class Logger{
    friend void initLogger(const std::string &info_filename);
public:
    Logger(log_rank_t log_rank):m_log_rank(log_rank){};
    ~Logger();
    static std::ostream &start(log_rank_t log_rank, const int line, const std::string &fun);


private:
    static std::ostream &getStream(log_rank_t log_rank);
    static std::ofstream m_info_file;
    log_rank_t m_log_rank;
};

#define LOG(log_rank) \
    Logger(log_rank).start(log_rank, __LINE__, __FUNCTION__)

#endif

logger.cpp文件 

#include <cstdlib>
#include <ctime>
#include "logger.h"

#include <windows.h>
std::ofstream Logger::m_info_file;


void initLogger(const std::string &info_filename){
    Logger::m_info_file.open(info_filename.c_str(), std::ofstream::app|std::ofstream::out);
}

std::ostream &Logger::getStream(log_rank_t log_rank){
    return m_info_file.is_open() ? m_info_file : std::cout;
}

std::ostream &Logger::start(log_rank_t log_rank, const int line, const std::string &fun){
    SYSTEMTIME        stLocal;
    GetLocalTime(&stLocal);
    std::string nowtime=std::to_string(stLocal.wYear)+'/'+std::to_string(stLocal.wMonth)+'/'+std::to_string(stLocal.wDay)+' '+std::to_string(stLocal.wHour)+':'+std::to_string(stLocal.wMinute)+':'+std::to_string(stLocal.wSecond)+'.'+std::to_string(stLocal.wMilliseconds);
    std::string logtype="I";
    if(log_rank == INFO){
        logtype="I";
    }else if(log_rank == WARNING){
        logtype="W";
    }else{
        logtype="E";
    }
    return getStream(log_rank) <<"["<< nowtime
                              << " " << logtype
                              << " " << fun
                              << " " << line <<"]:"
                              << std::flush;
}

Logger::~Logger(){
    getStream(m_log_rank) << std::endl << std::flush;

    if(m_log_rank == FATAL){
        m_info_file.close();
        abort();
    }

}

C++基于消息队列的多线程实现

实现消息队列的关键因素是考量不同线程访问消息队列的同步问题。 

std::lock_gurad 是 C++11 中定义的模板类  
通常用于管理某个锁(Lock)对象,方便线程对互斥量上锁，即在某个 lock_guard 对象的声明周期内，它所管理的锁对象会一直保持上锁状态；而lock_guard的生命周期结束之后，它所管理的锁对象会被解锁(注：类似 shared_ptr 等智能指针管理动态分配的内存资源 )。  
模板参数 Mutex 代表互斥量类型，例如 std::mutex 类型，它应该是一个基本的 BasicLockable 类型，标准库中定义几种基本的 BasicLockable 类型，分别 std::mutex, std::recursive_mutex, std::timed_mutex，std::recursive_timed_mutex 以及 std::unique_lock  

std::condition介绍  
当 std::condition_variable 对象的某个 wait 函数被调用的时候，它使用 std::unique_lock(通过 std::mutex) 来锁住当前线程。当前线程会一直被阻塞，直到另外一个线程在相同的 std::condition_variable 对象上调用了 notification 函数来唤醒当前线程。
std::condition_variable 提供了两种 wait() 函数。当前线程调用 wait() 后将被阻塞(此时当前线程应该获得了锁（mutex），不妨设获得锁 lck)，直到另外某个线程调用 notify_* 唤醒了当前线程。
在线程被阻塞时，该函数会自动调用 lck.unlock() 释放锁，使得其他被阻塞在锁竞争上的线程得以继续执行。另外，一旦当前线程获得通知(notified，通常是另外某个线程调用 notify_* 唤醒了当前线程)，wait() 函数也是自动调用 lck.lock()，使得 lck 的状态和 wait 函数被调用时相同。
在第二种情况下（即设置了 Predicate），只有当 pred 条件为 false 时调用 wait() 才会阻塞当前线程，并且在收到其他线程的通知后只有当 pred 为 true 时才会被解除阻塞。
