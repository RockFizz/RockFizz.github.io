导出不包含QT界面的C++dll

以QtCreator编辑器为例，新建项目起名withoutqtui，项目模板选Library>C++库->共享库
由于本demo只导出不包含QT界面的c++dll，
因此QtGui模块可以不选择，这里只选择了QtCore模块,下一步自动生成项目

withoutqtui.h: 

#ifndef WITHOUTQTUI_H
#define WITHOUTQTUI_H
#include "withoutqtui_global.h"

class WITHOUTQTUISHARED_EXPORT WithoutQtUI
{
public:
    WithoutQtUI();
};
#endif // WITHOUTQTUI_H

withoutqtui.cpp: 

#include "withoutqtui.h"
WithoutQtUI::WithoutQtUI()
{
}

注意withoutqtui_global.h这个文件，自动生成的内容如下： 

#ifndef WITHOUTQTUI_GLOBAL_H
#define WITHOUTQTUI_GLOBAL_H

#include <QtCore/qglobal.h>

#if defined(WITHOUTQTUI_LIBRARY)
#  define WITHOUTQTUISHARED_EXPORT Q_DECL_EXPORT
#else
#  define WITHOUTQTUISHARED_EXPORT Q_DECL_IMPORT
#endif

#endif // WITHOUTQTUI_GLOBAL_H

这里对Q_DECL_EXPORT做了重定义，那么Q_DECL_EXPORT关键字的含义是什么呢？  
QT文档里描述：   
根据您的目标平台上，Qt提供了包含必要的定义特殊的宏：
Q_DECL_EXPORT必须被添加到符号的编译一个共享库时使用的声明。
Q_DECL_IMPORT必须被添加到符号的编译使用共享库的客户机时使用的声明。  
那我们就理解成：Q_DECL_EXPORT宏主要为了导出这个类，生成.dll的同时生成.lib文件。提供给外部接口供其他类使用。

添加方法,编译输出c++库

写了一个add加法函数，修改如下,点击编译就可以获得需要的DLL和LIB文件了 

// withoutqtui.h:  
#ifndef WITHOUTQTUI_H
#define WITHOUTQTUI_H

#include "withoutqtui_global.h"

class WITHOUTQTUISHARED_EXPORT WithoutQtUI
{
public:
    WithoutQtUI(int a,int b);
    int Add();
    int a;
    int b;
};
#endif // WITHOUTQTUI_H

// withoutqtui.cpp:  
#include "withoutqtui.h"
WithoutQtUI::WithoutQtUI(int a,int b)
{
    this->a=a;
    this->b=b;
}
int WithoutQtUI::Add()
{
    return this->a+this->b;
}

要注意一点,在不同的编译环境下生成的dll只能被相同编译环境的程序调用

Qt使用C++库

  1. 如果在没有导入库文件(.lib)，而只有头文件(.h)与动态链接库(.dll)时，我们才需要显示调用(使用过程代码较多繁琐)
  2. 如果.h .lib .dll 三个文件都全的话，使用隐式调用更方便。
  3. 通常Windows下程序显示调用dll的步骤分为三步(三个函数)：LoadLibrary()、GetProcAdress()、FreeLibrary()

显示调用

在只提供头文件和Dll文件情况下

Qt提供了一个QLibrary类供显示调用，

  1. 把.h文件复制到程序当前目录下,把.dll/.so文件复制到程序的输出目录
其中，LoadLibrary() 函数用来载入指定的dll文件，加载到调用程序的内存中(DLL没有自己的内存！)  
GetProcAddress() 函数检索指定的动态链接库(DLL)中的输出库函数地址，以备调用  
FreeLibrary() 释放dll所占空间

完整加载

隐式调用

隐式调用比较方便，主要步骤如下：

  1. 把.h与.lib/.a文件复制到程序当前目录下，然后再把.dll/.so文件复制到程序的输出目录
  2. 在pro文件中，添加 .lib 文件的位置： LIBS+= -LD:\hitempt\api\ -lmyDLL
    添加包含头文件的位置： INCLUDEPATH += $$PWD\withoutQtUI
  3. 在宿主主程序中引入相应头文件#include “xx.h”，如果dll是c语言写的，则
    extern “C”{
    #include “xx.h”
    }
    来引入头文件

因为之前制作导出的dll是包含QtCore的，
所以为方便测试，这里新建项目模板是Applicat->Qt Console Application
项目起名qtconsoledemo: 

// qtconsoledemo.pro中新添加以下内容 
// 语法是LIBS+= -L直接打地址 -l直接打库名  
// 如我们有D:\QTCode\testVer\libs\libdwIme.a这个文件，则：
// LIBS += -LD:\QTCode\testVer\libs\ -ldwIme
LIBS += -L$$PWD\withoutQtUI\ -lwithoutQtUI
INCLUDEPATH += $$PWD\withoutQtUI
DEPENDPATH += $$PWD\withoutQtUI  

// 在项目下新建qtconsoledemo/withoutQtUI目录，
// 放入相应的头文件和.lib(.a)文件
// 本例放入qtconsoledemo/withoutqtui.h, qtconsoledemo/withoutqtui.lib, qtconsoledemo/withoutqtui_global.h  

// 把dll文件复制到程序的输出目录
// 本例放入qtconsoledemo/release/withoutQtUI.dll

隐式调用过程： 

#include <QCoreApplication>

#include <QDebug>
#include "withoutqtui.h"

int main(int argc, char *argv[])
{
    QCoreApplication a(argc, argv);
    qDebug()<<"hello wolrd!";
    WithoutQtUI mydll(2,3);
    qDebug()<< mydll.Add();
    return a.exec();
}
