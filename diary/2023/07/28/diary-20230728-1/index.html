在开发板上搭建linux系统并运行自己的程序

准备软硬件和文件

本文所需要的硬件平台为ARM-cortex A9内核的开发板，以iTOP-4412为例。  

准备软件需要以下几个：  
1. VMware(安装Linux虚拟机)  
2. Hyper Terminal(超级终端，用来与开发板通信)  
3. SSH Secure File Tranfer(用来主机与虚拟机之间通信与传递文件)  

准备文件需要以下几个：  
1. u-boot.bin(如果开发板可以正常启动，可以不需要这个)  
2. zImage  
3. ramdisk-uboot.img  
4. system.img  
5. fastboot工具

步骤：烧写系统进开发板

  1. 打开超级终端Hyper Terminal，根据相应的串口号连接开发板和电脑，
  2. 将前四个文件(u-boot.bin、zImage、ramdisk-uboot、system.img)拷贝到fastboot工具里面，开启开发板，即刻在超级终端上按回车进入boot模式，然后依次输入以下命令(每一条都是单独输入)： 
fdisk -c 0
fatformat mmc 0:1
ext3format mmc 0:2
ext3format mmc 0:3
ext3format mmc 0:4
fastboot
  1. 打开fastboot工具，并依次输入以下命令： 
fastboot.exe flash bootlaoder u-boot.bin #如果开发板能正常启动，可以不用烧录这个文件
fastboot.exe flash kernel zImage
fastboot.exe flash ramdisk ramdisk-uboot.img
fastboot.exe flash system system.img
fastboot -w
fastboot reboot
  1. 至此就完成了在开发板上搭建最小的Linux系统

编写程序放到开发板的linux系统里运行

在虚拟机里编写程序代码，交叉编译得到二进制文件
将可执行二进制拷贝到home下bin目录内

// 编写程序代码helloworld.c:  
#include <stdio.h>  
void main(){
    printf("Hello world!\n");
}
// 交叉编译得到二进制文件  
arm-none-linux-gnueabi-gcc -o helloworld helloworld.c -static  
// 将可执行二进制拷贝到home下bin目录内
cp -r helloworld /home/munilinux/system/bin/

重新编译得到新的system.img 

// 在目录/home/munilinux下进行编译  
make_ext4fs -s -l 314572800 -a root -L linux system.img system

将重新编译得到的新system.img按照上之前烧写系统进开发板，重新烧写进开发板（注意此时只运行烧写system.img即可）

烧写进去的helloworld程序在开发板的/bin目录下，目录下直接输入helloworld即可正常运行该程序

超级终端连接开发板并通信的一般步骤

用超级终端软件连接开发板之前，先要设置开发板的启动模式 

一般开发板上会有个开关，
拨至一侧（通常标识SDBOOT），系统从SD卡启动，常用于烧写系统或者从SD卡启动系统。  
拨至另一侧，系统从NandFlash，常用于正常启动系统

一般情况，5v电源适配器连接到开发板的5v输入插座上，开发板通过串口线将开发板上的串口如COM0与PC机（通常安装好串口转USB的驱动）的串口相连接。

打开超级终端，各种取消、否，新建连接，起名ttyS0,取消电话拨号调制解调器，在后续对话框可以选择”连接时使用”选择串口1”COM1”

最后一步，设置串口属性，必须选择”数据流控制”为”无”,否则只能看到输出而不能输入。设置”每秒位数(B)”为开发板串口波特率如115200，一般设置数据位0，奇偶校验无，停止位1.

所有参数设置好后，打开电源开关，终端出现系统启动画面，通过shell命令与开发板交互

SSH Secure File Transfer在window和linux之间传送文件

window安装SSHSecureShellClient_3.2.9_XiaZaiBa.zip后有两个可执行文件：一个是命令窗口（命令窗口客户端）用于登录linux进入shell操作，一个上传功能窗口。登录成功后点击”New filetransfer window”按钮打开文件传送界面。

如果无法登陆，ssh client报algorithm negotiation failed的原因是ssh升级后，为了安全，默认不再采用原来一些加密算法，我们手工添加进去即可。添加方法如下 

sudo vim /etc/ssh/sshd_config  
进入编辑状态，在配置文件的末尾添加如下内容  
Ciphers aes128-cbc,aes192-cbc,aes256-cbc,aes128-ctr,aes192-ctr,aes256-ctr,3des-cbc,arcfour128,arcfour256,arcfour,blowfish-cbc,cast128-cbc  
MACs hmac-md5,hmac-sha1,umac-64@openssh.com,hmac-ripemd160,hmac-sha1-96,hmac-md5-96  
KexAlgorithms diffie-hellman-group1-sha1,diffie-hellman-group14-sha1,diffie-hellman-group-exchange-sha1,diffie-hellman-group-exchange-sha256,ecdh-sha2-nistp256,ecdh-sha2-nistp384,ecdh-sha2-nistp521,diffie-hellman-group1-sha1,curve25519-sha256@libssh.org  

退出编辑状态 :wq!保存修改  
重启ssh服务 sudo systemctl restart ssh

ubuntu下linux内核裁剪的流程

LINUX系统一直以来都备受关注，其中Ubuntu是其中最受欢迎的系统之一。它的开源性质、免费使用以及高度可定制性，让用户可以自由地对系统进行修改和定制。但是，有些用户可能会想知道Ubuntu系统是否可以进行裁剪，以便更好地满足他们的需求。在本文中，我们将探讨这一问题。
Ubuntu系统是基于Debian系统开发的，它采用了GNOME桌面环境，包含了大量的预装软件。这些软件虽然功能强大，但是对于一些用户来说可能并不需要或者用不上。因此，用户可能会想要进行裁剪，以便更好地满足他们的需求。在实际操作中，Ubuntu系统是可以进行裁剪的。用户可以使用命令行工具或者图形化界面来卸载不需要的软件包，以减少系统的体积和资源占用。此外，用户还可以自己编译内核，以去除一些不需要的模块和驱动程序，从而进一步减小系统的体积和资源占用。

裁剪系统示范文章A

1.卸载不需要的软件包

用户可以使用命令行工具apt-get或者图形化界面Synaptic Package Manager来卸载不需要的软件包。
命令行方式如下： 

sudo apt-get remove package_name

图形化界面方式如下： 

打开Synaptic Package Manager，搜索需要卸载的软件包，右键选择“Mark for Removal”，点击“Apply”进行卸载。

2.编译内核

用户可以通过编译内核来去除一些不需要的模块和驱动程序。 

下载内核源代码
sudo apt-get source linux-image-$(uname -r)  
进入源代码目录  
cd linux-$(uname -r)  
配置内核,在配置界面中，去除不需要的模块和驱动程序，保存并退出  
make menuconfig  
编译内核  
make
安装内核  
sudo make install  
重启系统，选择新编译的内核启动

裁剪系统示范文章B

安装所需工具和依赖项 

$ sudo apt-get update
$ sudo apt-get install build-essential libncurses-dev bison flex libssl-dev libelf-dev

下载内核源代码并解压缩,您可以从官方网站下载最新版本的内核源代码，或者使用以下命令下载当前安装的内核版本 

$ sudo apt-get source linux-image-$(uname -r)

进入已解压缩的目录 

$ cd linux-$(uname -r)

打开配置界面 

$ make menuconfig

开始裁剪，在配置界面中，您可以选择启用或禁用特定功能、驱动程序和模块。建议仔细阅读每个选项并根据需要进行调整。
完成配置后，请保存更改并退出菜单。此时将生成一个.config文件。

开始编译内核，运行以下命令编译新的内核映像文件 

$ make -j $(nproc)  
其中-nproc选项指定要使用的CPU线程数

最后，更新GRUB引导程序配置以包含新内核。运行以下命令 

$ sudo update-grub

现在，您可以重新启动计算机并选择新内核。如果一切正常，您已成功裁剪了Linux内核

如何更换默认的系统启动内核

查看系统目前正在使用的内核: uname -r
查看系统可用的内核: dpkg --get-selections | grep linux-image
其中显示 install 的就是当前系统可用的内核  

修改/etc/default/grub文件  
sudo gedit /etc/default/grub
我们不修改 GRUB_DEFAULT 参数的值，而是添加如下两条命令  
GRUB_SAVEDEFAULT=true
GRUB_DEFAULT=saved
接着注释掉 GRUB_TIMEOUT_STYLE 这一段代码，设置GRUB_TIMEOUT值为30:  
# GRUB_TIMEOUT_STYLE=hidden
GRUB_TIMEOUT=30
这样做的目的是使得开机前留下30秒的时间进入 grub 选择界面，接着更新 grub 配置  
sudo update-grub
重启系统  
sudo reboot
开机之前的 grub 界面中，选择 Advantages for ubuntu 选项，接着选择自己想要的内核版本进入便可，系统会默认选择你本次的启动内核为默认内核版本  

开机后依然使用uname -r查看系统内核版本，  
接着需要注释掉GRUB_TIMEOUT=30这一段代码，设置GRUB_TIMEOUT_STYLE值为hidden，不然下次开机还会弹出 grub 界面：
更新一下配置  
sudo update-grub  
完成

更清晰的局域网udp广播示例

服务端和接收端共用一个类，初始化作为服务端的udp在不断发送消息，初始化作为接收端的udp每隔2秒接收一次消息
一个udp客户端发送的数据报，在同一网络范围内其他所有的udp客户端都可以收到。其支持IPV4广播

  1. 新建qt项目DemoUDPClient和项目DemoUDPServer，工程模板均为Application->Qt Console Application 
各自项目的**.pro中添加Network模块  
QT += core network
  1. 各自项目均添加c++类udpserverandclient 
//udpserverandclient.h  
#ifndef UDPSERVERANDCLIENT_H
#define UDPSERVERANDCLIENT_H

#include <QObject>
#include <QUdpSocket>
#include <QTimer>

#include <QHostInfo>
#include <QDataStream>
#include <QDateTime>
#include <QHostAddress>

class udpserverandclient :public QObject
{
    Q_OBJECT
public:
    explicit udpserverandclient(QObject *parent=0);
    void setServerInit();
    void setClientInit();

private slots:
    void onServerSendBroadCast();
    void onPendingDatagrams();
private:
    QUdpSocket udpSocketServer;
    QUdpSocket udpSocketClient;
    QTimer timer;
};

#endif // UDPSERVERANDCLIENT_H

//udpserverandclient.cpp  
#include "udpserverandclient.h"

udpserverandclient::udpserverandclient(QObject *parent):QObject(parent)
{
}

void udpserverandclient::setServerInit(){
    connect(&timer,SIGNAL(timeout()),this,SLOT(onServerSendBroadCast()));
    timer.start(2000);
}

void udpserverandclient::setClientInit(){
    udpSocketClient.bind(13998);
    connect(&udpSocketClient,SIGNAL(readyRead()),this,SLOT(onPendingDatagrams()));
}
void udpserverandclient::onServerSendBroadCast(){
    QString hostName=QHostInfo::localHostName();
    QString now=QDateTime::currentDateTime().toString("yyyy-MM-dd hh:mm:ss");
    QString msg=QString("%1 on %2").arg(hostName).arg(now);
    udpSocketServer.writeDatagram(msg.toLocal8Bit(),QHostAddress("192.168.157.255"),13998);
    // 当作服务端不停的发送udp数据
}

void udpserverandclient::onPendingDatagrams(){
    QByteArray datagram;
    do{
        datagram.resize(udpSocketClient.pendingDatagramSize());
        udpSocketClient.readDatagram(datagram.data(),datagram.size());

        QString fromUDP=datagram;
        QString selfName=QHostInfo::localHostName();
        QString msg=QString("%1 fromUDP, localPC name is %2").arg(fromUDP).arg(selfName);
        qDebug()<<msg; //client通过监听udp广播得到的数据
    }while(udpSocketClient.hasPendingDatagrams());
}
  1. 当服务端使用的项目main.cpp 
#include <QCoreApplication>
#include "udpserverandclient.h"
int main(int argc, char *argv[])
{
    QCoreApplication a(argc, argv);
    udpserverandclient u;
    u.setServerInit();
    return a.exec();
}
  1. 当客户端使用的项目main.cpp 
#include <QCoreApplication>
#include "udpserverandclient.h"
int main(int argc, char *argv[])
{
    QCoreApplication a(argc, argv);
    udpserverandclient u;
    u.setClientInit();
    return a.exec();
}

ubuntu系统操作实践

配置静态IP

my@mypc:~$ sudo vim /etc/network/interfaces  
auto ens33
# iface ens33 inet dhcp
iface ens33 inet static
address 192.168.40.92
netmask 255.255.255.0
gateway 192.168.40.1

dns-nameserver 8.8.8.8  
保存退出后，重启网络
my@mypc:~$ sudo /etc/init.d/networking restart

自动分配IP 

my@mypc:~$ sudo vim /etc/network/interfaces  
auto lo
iface lo inet loopback
保存退出后，重启网络
my@mypc:~$ sudo /etc/init.d/networking restart

常用指令

查看SSH服务状态 sudo service ssh status  
查看内存使用情况 free -lh
查看设备存储情况(可以观察到U盘拔插) df -lh

系统监控，类似window中的进程管理器，可以看出cpu内核、mem大小、所有进程pid等等
sudo apt-get install htop
查看sudo htop退出按键q  

原始命令，逐一查看进程 ps aux | more

流量监控工具iftop  
sudo apt-get install iftop
通过ifconfig得到网卡名字如eth0,  
然后对该网卡流量实时监控 sudo iftop -i eth0 -n -P
按键q退出监控  

top命令可以用于实时监控进程的系统资源使用情况，比如CPU、内存、I/O等。它会周期性地更新屏幕上的进程列表，并将最具 CPU 使用率的进程放在最上方
