文件结构

每个c++/c程序通常分为两个文件，一个用于保存程序的声明(declaration)，称为头文件。另一个文件用于保存程序的实现(implementation)，称为定义文件(definition)。
c++/c的头文件以”.h”后缀，c程序定义文件以”.c”后缀，c++定义文件以”.cpp”后缀

头文件结构

1.头文件开头处版权和版本声明
2.预处理块
3.函数和类结构声明

1. 为防止头文件被重复引用，应该用#ifndef、#define、#endif产生预处理块。  
2. 用#include <filename.h>引用标准库的头文件。（编译器从标准库目录开始搜索）。  
3. 用#include "filename.h"引用非标准库的头文件。（编译器从用户工作目录开始搜索）。
4. 头文件只存放声明，不存放定义。 
5. 变量一般不要定义在.h文件中 

#ifndef <标识> 
#define <标识> 
...... 
...... 
#endif 
<标识>在理论上来说可以是自由命名的，但每个头文件的这个“标识”都应该是唯一的。标识的命名规则一般是头文件名全大写，前后加下划线，并把文件名中的“.”也变成下划线，如：stdio.h 

#ifndef _STDIO_H_ 
#define _STDIO_H_ 
#include <math.h> //引用标准库头文件  
#include "myheader.h" // 引用非标准库头文件  
void FunctionA1(); // 函数声明  
class Box{

}；             // 类结构声明           
#endif

定义文件结构

1.定义文件开头处版权和版本声明
2.对头文件的引用
3.程序实现体，包括数据和代码 

#include "graphics.h"  //引用头文件  

void FunctionA1(){

}                   // 函数实现体  

void Box::Draw(){

}                   // 类成员函数实现体

头文件的作用

通过头文件调用库函数，很多场合，源代码不便于向用户公布，只要向用户提供头文件和二进制的库即可。用户只需要按照头文件中的接口声明来调用库功能，而不必关系接口如何实现，编译器会从库中提取相应的代码。

头文件能加强类型安全检查，如果某个接口被实现或被使用时，其方式与头文件中声明不一致，编译器就会指出错误。

目录结构

如果一个软件头文件数目比较多(如超过十多个)，通常应将头文件和定义文件分别放在不同的目录以便维护。例如将头文件保存于include目录，将定义文件放于source目录。

修饰符*和&的位置

修饰符*和&应紧靠变量名 

char *name;  
int *x,*y,z;  // x,y是指针，z不是指针

常用命名规则

类名和函数名用大写字母开头
变量和参数名用小写字母开头
常量名全用大写字母，用下划线分割
静态变量加前缀s_
全局变量加前缀g_
类的数据成员加前缀m_

语句

布尔变量与零值比较

假设布尔变量名为flag，与零值比较的标准语句如下  
if(flag)  // 表示flag为真  
if(!flag) // 表示flag为假  
不可将布尔变量直接与TRUE、FALSE或者1、0进行比较  
其他用法属于不良风格，例如以下  
if(flag==TRUE)  
if(flag==1)
if(flag==FALSE)  
if(flag==0)

整型变量与零值比较 

整型变量用"=="或"!="直接与0比较  
if(value==0)  
if(value!=0)

浮点变量与零值比较
不可将浮点变量用”==”或”!=”与任何数字比较
千万要留意，无论是float或double，都有精度限制，不能使用”==”或”!=”与任何数字比较，应设法转化为”>=”或”<=”

指针变量与零值比较

指针的零值是NULL,应当将指针变量用”==”或”!=”与NULL比较

c++/c循环语句中，for语句频率最高，while次之，do语句很少用。

提高循环体效率的基本方法是降低循环体的复杂性。 

1. 多重循环中，应当将最长的循环放在最内层，最短的循环放外层，以减少CPU跨切循环层的次数  
2. 如果循环体内存在逻辑判断，并且循环次数很大，宜将逻辑判断移到循环体外面。

常量

C用#define定义常量，C++除了#define外还可以用const来定义常量

#define MAX 100 // C语言的宏常量  
const int MAX=100; // C++的const常量
const float PI=3.14159; // C++的const常量
const常量具有数据类型而宏常量没有，C++中const常量可以完全取代宏常量

需要对外公开的常量放在头文件里。
不需要对外公开的常量放在定义文件里。

类的const数据成员只在某个对象生存期内是常量，不同对象其const数据成员值可不相同。
不能在类声明中初始化类的const数据成员，类的const数据成员只能在类构造函数的初始化表中初始化。 

// 类的声明
class A{
    ...  
    A(int size); //构造函数  
    const int SIZE;
};
// 类的构造函数的初始化表  
A::A(int size):SIZE(size){
    ...
}

A a(100); // 对象a的SIZE值为100  
A b(200); // 对象b的SIZE值为200

对整个类都恒定的常量，通过枚举常量实现
枚举常量不会占用对象的存储空间，在编译时被全部求值，缺点是隐含数据类型是整数，最大值有限且不能表示浮点数（如PI=3.14159） 

class A{
    ...  
    enum{
        SIZE1=100,
        SIZE2=200
    };
    int array1[SIZE1];
    int array2[SIZE2];
}

函数设计

函数接口两个要素是参数和返回值，C语言中参数和返回值的传递方式有两种：值传递和指针传递。C++语言多了引用传递。 

如果参数是指针且仅作输入用，应在类型前加const，防止该指针在函数体内被意外修改。  
void StringCopy(char *strDestination, const char *strSource);

如果参数以值传递方式传递对象，宜改用"const &"，省去临时对象的构造和析构过程提高效率。

引用与指针的比较

引用相当于被引用物的别名，对引用的任何操作就是对被引用物的操作。 

int m;
int &n=m;  
n是m的一个引用，m是被引用物  
n既不是m的拷贝，也不是指向m的指针，n就是m它自己。  
1. 引用被创建时必须同时被初始化（指针则可以在任何时候被初始化）  
2. 不能有NULL引用（指针可以是NULL）  
3. 引用一旦被初始化就不能改变引用关系（指针可以随时改变所指对象）

值传递、指针传递、引用传递

以下值传递：Func1函数体内x是外部变量n的一份拷贝，改变x值不会影响n  
void Func1(int x){
    x=x+10;
}
int n=0;
Func1(n);  

以下指针传递：Func2函数体内x是指向外部变量n的指针，改变指针的内容将导致改变n值   
void Func2(int *x){
    (*x)=(*x)+10;
}
int n=0;
Func2(&n);

以下引用传递：Func3函数体内x是外部变量n的引用，引用和被引用物是同一个东西，改变x等于改变n     
void Func3(int &x){
    x=x+10;
}
int n=0;
Func2(n);

实际上，引用能做的任何事用指针也能够做，为什么需要引用？ 答案是“用恰当的工具做恰如其分的工作”
指针能毫无约束的操作内存中任何东西，比较危险。如果的确只是需要借用一下某对象的别名就用引用而不要用指针，以免发生意外。

内存管理

内存分配方式有三种：

  1. 静态存储区分配：内存在程序编译时已经分配好，这块内存在程序整个运行期间都存在，如全局变量，static变量。
  1. 栈上创建：执行函数时，函数内局部变量的存储单元都可以在栈上创建，函数执行结束时这些存储单元自动被释放，栈内存分配运算内置于处理器指令，效率高但分配的内存空间容量有限。
  1. 堆上分配：亦称动态内存分配，程序运行时用malloc或new申请任意大小内存，自行负责何时用free或者delete释放内存。申请内存后应立即检查指针值是否为NULL，防止使用指针值为NULL的内存

常见内存错误：内存分配未成功，却使用了它

使用内存前应检查指针是否为NULL,如果指针p是参数，使用时应if(p!=NULL)防错处理

常见内存错误：内存分配成功，但是未初始化 

不要忘记为数组和动态内存赋初值

常见内存错误：内存分配成功且已经初始化，但操作越过了内存边界 

避免数组或指针的下标越界

常见内存错误：忘记释放内存，造成内存泄漏 

malloc与free使用次数一定要相同，new/delete同理。

常见内存错误：释放内存后却继续使用它 

return不要返回指向栈内存的指针或引用。  
使用free或delete释放内存后需要将指针设为NULL，预防野指针。
野指针的成因主要有两种：指针变量没有被初始化，指针被free/delete后没有被置为NULL.

指针和数组的对比

c/c++中指针和数组在不少地方可以替换使用。
数组要么在静态存储区创建（全局数组），要么在栈上创建。数组名对应一块内存，其地址和容量在生命期内保持不变，数组只有内容可以改变。
指针可随时指向任意类型的内存块，远比数组灵活且危险。

内容修改 

如下所示，字符数组a的容量是6个字符 内容为hello\0  
a的内容可以改变，如a[0]='X'  
指针p指向常量字符串"world"(位于静态存储区，内容为world\0)  
p[0]='x'，编译器从语法来说并无不妥，但运行时企图修改常量字符串内容而报错  

char a[]="hello";
a[0]='X';
char *p="world";
p[0]='x';

内容复制与比较

不能对数组名进行复制和比较 

p=a并不能将a的内容复制指针p，而是将a的地址给了p，想复制a的内容，  
先用库函数malloc为p申请容量为strlen（a）+1个字符的内容，再用strcpy进行字符串复制。  
同理，if(p==a)比较的是地址而不是内容，应用strcmp来比较

计算内存容量

sizeof可以计算数组的容量（字节数）。
c++/c语言无法知道指针所指内存容量，除非在申请内存时记住它。

当数组作为函数参数传递时，该数组自动退化为同类型指针 

char a[]="hello world";
char *p=a;
cout << sizeof(a)<< endl; // 12字节
cout << sizeof(p)<< endl; // 4字节

当函数参数是一个指针时，不要用该指针去申请动态内存。
非得用指针参数申请内存时，请改用”指向指针的指针” 

void GetMemory(char **p,int num){
    *p=(char*)malloc(sizeof(char)*num);
}
void Test(){
    char *str=NULL;
    GetMemory(&str,100);
    strcpy(str,"hello");
    free(str);
}

new/delete申请动态内存使用后释放 

Obj *a=new Obj;
...
delete a;

int *b=new int[100];
...
delete b;
