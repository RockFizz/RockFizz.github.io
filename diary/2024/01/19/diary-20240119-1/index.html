MFC(MicrosoftFoundationClasses)

微软公司提供一个类库，以C++形式封装Windows的API,并且包含一个应用程序框架。其中的类包含大量Windows句柄封装类和Windows内建控件及组件封装类。

Win32程序基本概念


windows程序分类源码和UI资源两部分，二者最后被编译器整合为一个完整的exe。

vs2015 新建项目 -> Visual C++ -> MFC应用程序    
进入 MFC应用程序向导，应用程序类型选择"基于对话框" 其它属性看看设置，点击确定完成  

在资源视图（资源视图如关闭可在视图->其他窗口->资源视图重新打开）下选中对话框资源，双击打开设计界面，从工具箱视图拖拽控件，调整布局  

比如，从工具箱拖动一个按钮控件到对话框合适位置，选中该按钮，右键选择属性，在属性窗口，设置"ID"为唯一值，
双击按钮控件，会进入自动生成的消息处理函数处添加  
MessageBox(_T("Hello World"), _T("Message"), MB_OK);

MFC 中调试程序并打印信息，
在需要打印信息的地方，使用 TRACE 宏输出调试信息，例如
TRACE(_T("My debug message.\n"));
在菜单栏中选择 "调试" -> "窗口" -> "输出"。（如果 "输出" 窗口没有显示，可以通过 "工具栏" -> "输出" 打开）
在 "输出" 窗口中，可以看到 TRACE 宏输出的调试信息。

MFC中移动按钮的位置和调整大小
1. 获取按钮句柄（句柄就是指针的意思）  
CButton* pButton = (CButton*)GetDlgItem(IDC_MY_BUTTON);  
2. 移动按钮的位置  
pButton->MoveWindow(100, 100, 150, 30);


VS2015显示代码行数："工具" -> "选项" -> "文本编辑器" -> "C/C++" ->  "行号" 打勾

资源视图下，右键可以"添加资源..."可以添加各种资源  
想用代码控制这些控件和页面，第一步需要添加类  
选择整个对话框，右键选择，添加类，  
然后在弹出的对话框中，填写类名，选择基类，基类可以不改变比如CDialog，默认就行，

右键还有"类向导"，辅助生成类

增加菜单命令

1 打开MFC应用程序的资源视图，找到菜单资源。  
2 双击菜单资源，将会打开菜单编辑器。  
3 在菜单编辑器中，你可以添加一个新的菜单项。只需选择你想要添加新菜单项的位置，然后按Enter键。  
4 在新菜单项的属性中，你可以设置菜单项的ID和标题。  
5 接着，你需要为新菜单项添加消息处理函数。你可以通过点击新菜单项，右键点击"添加事件处理程序"，将会自动为你创建一个新的消息处理函数。  
（注意，菜单项属性中Popup要改成False后，该菜单项才可以编辑ID）
6 在新创建的消息处理函数中，你可以使用MessageBox函数来创建弹框。例如：
void CYourApp::OnNewMenuItemClicked()
{
    MessageBox(_T("新菜单项被点击了！"), _T("提示"), MB_OK);
}
7 编译并运行你的应用程序，当用户点击新添加的菜单项时，弹框将会显示出来

点击菜单弹出自定义对话框

1. 资源视图，右键添加资源，新建一个自定义对话框，从工具箱里可以拖拽组件来自定义这个对话框的内容
2. 右键对话框，"添加类...",新建自定义对话框类CMyDialogS
3. 切换到资源视图，选中目标组件右键"添加事件处理程序"，在刚才创建的对话框类中添加相应的消息处理函数
4. 在要弹出对话框的位置，引入自定义对话框类的头文件
#include "./CMyDialogS.h"  
CMyDialogS dig;
dig.DoModal();

添加右键菜单

1. 资源视图，右键添加资源，新建菜单，右键菜单是第一个菜单的子菜单项内容（参考上面内容添加菜单响应事件处理），因为我们后面有个.GetSubMenu(0);
2. 资源视图，选中要弹出右键菜单的目标对话框，在其头文件中添加一个成员变量，用于保存弹出菜单的对象   
CMenu m_popupMenu;
3. 资源视图，选中要弹出右键菜单的目标对话框，右键"类向导..."，切换到"消息"tab，选择"WM_CONTEXTMENU"消息，点击"添加处理程序"，点击"确定"

MFC程序发布

需求：写好的MFC程序，生成exe文件后，可以在未安装VS的电脑中直接运行
最简单的方法即使用静态库，这样生成的MFC程序拷贝到其他电脑，无需安装即可直接运行，具体设置如下： 

打开已经写好的MFC程序；  
选择“项目 ---> 属性 ---> 配置属性 ---> 常规”；  
在"MFC的使用"项中选择“在静态库中使用MFC”  
选择“确定”，重新生成exe文件  

选择“项目 ---> 属性 ---> 配置属性 ---> 常规”；注意"字符集"的设置保持Release和Debug的配置要一致，否正会出现Debug能编译而Release编译不通过的现象，因为字符集不同，导致字符串处理时一个是const char*一个是const wchar_t，一个是LPCWSTR一个是另外一种

重新release，如果编译失败看vs原因，可能哪里字符串需要(LPCWSTR)强制转换  
重新编译成功后，Release/*.exe和Release/*.pdb一起赋值到其它电脑即可独立运行

Debug模式下，
int bufSize=12;
TRACE("this is %d",bufSize);
一，TRACE函数的输出是在Output窗口的Debug选项下显示； 
二，只有在DEBUG版本调试时才会有输出，如果是在Release版本调试或者运行程序时，将不会看到输出。

MFC通过tcp发送和接收信息

MFC是一个用于Windows应用程序开发的C++类库。要通过TCP发送和接收信息，可以使用MFC提供的套接字类（CAsyncSocket或CSocket）来实现。

首先，在MFC应用程序中包含必要的头文件  
#include <afxsock.h>

然后，创建一个套接字对象，并设置其相关属性，如IP地址、端口号等
CAsyncSocket sock;
if (!sock.Create())
{
    // 套接字创建失败
    return;
}
// 设置服务器IP地址和端口号
CString serverIP = "192.168.1.100";
UINT serverPort = 12345;
if (!sock.Connect(serverIP, serverPort))
{
    // 连接服务器失败
    return;
}

要发送信息，可以调用套接字的Send函数：  
CString message = "Hello, server!";
int len = message.GetLength();
if (sock.Send(message, len) == SOCKET_ERROR)
{
    // 发送失败
    return;
}

要接收信息，可以使用套接字的Receive函数：  
char buffer[1024];
int bufferSize = sizeof(buffer);
int bytesRead = sock.Receive(buffer, bufferSize);
if (bytesRead == SOCKET_ERROR || bytesRead == 0)
{
    // 接收失败或连接关闭
    return;
}
// 处理接收到的数据
buffer[bytesRead] = '\0';
CString receivedData(buffer);

最后，记得在适当的时候关闭套接字连接：  
sock.Close();

这只是基本的示例代码，实际应用中还需要进行错误处理、异常处理等。另外，确保在使用套接字之前已经初始化了Windows Sockets库（WSAStartup函数）。
