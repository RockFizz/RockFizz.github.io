预备知识

+

C++三件法宝，融合了三种不同的编程方式

  1. 继承C语言高效、简洁、快速、可移植

标准的C语言代表着过程性语言，提供了基本类型，运算符，控制结构和语法规则，能生成简洁快速运行的程序。

标准C语言提供了处理硬件的能力，如通信端口和磁盘驱动器。

  1. 面向对象编程

C++在C语言基础上添加的类代表的面向对象语言（对象、类、封装、数据隐藏、多态、继承）

  1. 泛型编程(利用模板特性)

C++模板使其支持泛型编程

诞生

操作系统是能够管理计算机资源，处理计算机与用户交互的一组程序。20世纪70年代，开发Unix操作系统的贝尔实验室需要一种语言，能生成简洁快速的程序并有效控制硬件。

在这之前，传统程序员使用汇编来满足这个需求（能生成简洁快速的程序并有效控制硬件）

低级语言（如汇编语言）直接操作硬件（直接访问CPU寄存器和内存单元），因此汇编是针对特定计算机处理器的，要讲汇编程序移植到另一台计算机上，必须使用不同的汇编语言重新编写程序。

然而Unix是为不同的计算机（平台）上工作而设计的，这意味着它不针对特定硬件，是一种高级语言

出现编译器，它是一种特殊程序（将高级语言翻译成特定计算机硬件的内部语言），通过对每个平台使用不同的编译器，来使用同一个高级语言程序。

贝尔实验室希望有一种语言能将低级语言的效率、硬件访问能力和高级语言的通用性移植性融合在一起，于是开发出C语言 

计算机语言要处理两个概念，数据和算法。数据是程序使用和处理的信息，算法是程序使用的方法。  

过程性编程首先确定计算机应采取的操作，然后使用编程语言实现这些操作。程序命令按一系列流程生成特定的结果。  

随着程序规模的扩大，早期语言（basic和fortran）会遇到组织方面的问题，如经常使用分支语句导致几乎不可能通过阅读来理解它。  

科学家开发了一种更有序的编程方式---结构化编程，将分支限制为一小组行为良好的结构。（for循环、while循环、dowhile循环、ifelse语句）  

另一个c语言的原则是自顶而下的设计：  
C语言的理念是将大型程序分解为小型便于管理的任务，如果其中一项任务仍然过大，则将它分解为更小的任务。这一过程一直持续下去，直到将程序划分为小型易于编写的模块。  

结构化编程反映了过程性编程的思想，根据执行的操作来构思一个程序

结构化编程思想提高了程序的清晰度可靠性并使之便于维护。但在编写大型程序时仍面临挑战。为应付挑战，OOP提供新的方法。

与过程性编程强调算法不同，面向对象编程(OOP)强调数据。

C++中类是一种规范，它描述了一种新型数据格式，对象是根据这种规范构造的特定数据结构。

通常，类规定了可使用哪些数据来表示对象，以及可以对这些数据执行哪些操作。

OOP程序设计首先设计类，准确表示程序要处理的东西。然后设计一个使用这些类的对象的程序，从低级组织（类）到高级组织（程序）的处理过程叫做自下而上的编程。 

OOP不仅将数据和方法合并成类定义  
还有助于创建可重用的代码。  
信息隐藏可以保护数据，使其免遭不适当的访问。  
多态能为运算符和函数创建多个定义，通过编程上下文确定使用哪个定义。  
继承让您能使用旧类派生出新类

OOP使程序员能轻松使用已有的类，厂商提供大量有用的类库，C++真正的优点是：可以方便的重用和修改现有经过仔细测试过的代码。

泛型是C++支持的另一种编程模式，与oop目标相同，都是为了重用代码和抽象通用概念。
不过OOP强调的是编程的数据方面，而泛型强调的是独立于特定数据类型。

OOP是管理大型项目的工具，泛型提供了执行常见任务的工具（如数据排序或链表合并）

泛型指创建独立于类型的代码。

C++的数据表示有多种类型，整数、小数、字符、字符串、用户定义的多种类型组成的复合结构。通常必须为每种类型创建一个排序函数，泛型对语言的扩展，可以只编写一个泛型（不指定具体是哪种类型）函数，并将其用于各种实际类型，C++模板提供了这种机制。

贝尔实验室之所以在C语言基础上创建c++，是因为c语言简洁适合系统编程，且与unix操作系统紧密联系。C++在C语言基础上加入OOP特性和对C的泛型编程支持。因此C++是c语言的超集，这意味着任何有效的c程序都是有效的c++程序。他们之间存在差异但无足轻重。

C++程序可以使用已有的c软件库。库是编程模块的集合，可以从程序中被调用，库对很多编程问题提供了可靠的解决方法。因此能节省大量的时间和工作量，有利于c++的广泛传播。

OOP部分赋予C++将问题涉及概念联系起来的能力，C部分赋予C++紧密联系硬件的能力。这种能力的结合造成了C++的广泛传播。

C++获得一定成功后，添加了模板，使得泛型编程成为可能，在模板特性被使用和改进后人们才逐渐认识到，模板和OOP同样重要甚至比OOP更重要。

可移植性和标准

假设你为操作系统为win2000的老奔腾PC编写了一个c++程序，而使用人员决定使用不同的操作系统（如Mac或Linux和处理器（如SPARC处理器）使用它，该程序是否能在新平台运行？

当然，必须使用为新平台设计的C++编译器对程序进行重新编译即可。

是否需要修改已编写好的c++代码呢？

如果在不修改代码的情况下，重新编译程序后程序运行良好。则成该程序是可移植的。

可移植性的两个障碍：硬件和语言差异

C++在不断发展，其为了消除语言差异的国际标准也在更新。

ANISI C标准不仅定义了C语言，还定义了一个ANSI C实现必须支持的标准C库。C++也使用这个库，（标准C库或标准库）还提供了C++标准类库。

C++98标准，C++11标准

创建程序

编写一个C++程序后如何运行起来？具体步骤取决于计算机环境和使用的C++编译器。大体如下：

  1. 使用文本编辑器编写程序，保存到文件中，这就是程序的源代码。
  2. 编译源代码。意味着运行另一个程序将源代码翻译为本主机使用的内部机器语言。包含了翻译后的程序的文件，就是程序的目标代码。
  3. 将目标代码和其他代码链接起来。（例如C++程序常用库）链接指的是将目标代码同使用的函数的目标代码以及一些标准的启动代码组合起来，生成程序的运行阶段版本。包含该最终产品的文件称为可执行代码。 
源代码  
编译器  
目标代码  
链接程序（添加启动代码和库代码）
可执行代码

为源代码文件命名时必须使用正确的后缀，.C是有效的UNIX C++源代码文件名 小写的.c是标准C才使用的

总结

随着计算机功能越来越大，计算机程序越来越复杂，为应对挑战，计算机语言也得到改进使编程过程更简单，C语言新增诸如控制结构和函数特性，以便更好控制程序流程，支持结构化模块化程度更高的方法。C++增加面向对象编程和对泛型编程的支持，有助于提高模块化和创建可复用的代码。

C++流行导致大量基于各种计算平台的C++实现，ISOC++标准为确保众多实现的相互兼容提供了基础，这些标准规定了语言必须具备的特性，行为，标准库函数，类和模板。目的在于实现C++在不同计算平台和实现之间的可移植性。

C++中每条完整的指令称为语句，所有语句以分号结束。
函数头位于函数名前面的部分叫做函数返回类型，位于函数名后面的部分叫参数列表。
main函数通常并不从程序其他部分调用，而是被启动代码调用，启动代码由编译器添加到程序中，是程序和操作系统之间的桥梁
c++程序必须包含一个名为main()的函数,注意大小写。
C++程序运行时从main()函数开始执行 

例外情况：window编程中编写动态链接库DLL模块  
因为DLL模块不是独立的程序，所以不需要main()函数

如果程序需要使用C++输入或输出工具，请提供以下两行代码 

#include <iostream>  
using namespace std;

如果编译器不接受这几行代码，说明它没有遵守标准C++98  

预处理器将iostream文件的内容添加到程序里，这是一种典型的预处理操作：  
在源代码被编译之前，替换或添加文本。  

实际上，iostream文件的内容将取代程序中#include <iostream>代码行的位置。  
原始文件没有被修改，而是将源代码文件和iostream组合成一个复合文件，编译的下一阶段将使用这个复合文件

类描述了一种数据类型的全部属性（包括可使用它执行的操作）
对象是根据这些描述创建的实体
类描述指定了可对类对象执行的所有操作。要对特定对象执行这些允许的操作，需要给对象发送一条消息。
C++提供两种发消息的方式，一种是使用类方法（本质是函数调用），另一种是重定义运算符（cin和cout采用这种）
使用重新定义的>> 和 << 发送消息给 cout 和 cin

函数用于创建C++程序的模块，C++函数分两种，有返回值和没有返回值的

使用有返回值的函数 

标准C/C++库包含一个sqrt()函数，返回平方根  
double x;
x =sqrt(6.25);  返回2.5赋值给x

sqrt()函数的函数原型：  
double sqrt(double);
原型结尾;表示它是一个原型，而不是函数头。如果省略;编译器将会把这样解释为函数头并要求后续接着提供函数体  

提供函数原型有两种方法：  
1. 在源代码中输入函数原型；
2. 包含头文件，其中定义了原型；

使用函数之前，C++编译器必须知道函数的参数类型和返回值类型。
缺少这些信息，编译器将不知道如何解释返回值，C++使用函数原型语句提供这些信息。
C++程序应该为程序中使用的每个函数提供原型。
函数原型之于函数就像变量声明之于变量

C++库函数存储在库文件中，标准C库提供140多个预定义函数。
每个C++程序必须有一个main()函数
对于库函数，在使用前必须提供原型，通常把原型放到main()定义前
对于自定义函数，通常把原型放到main()定义前，通常把函数体源代码放到main()定义后

总结

C++程序由一个或多个被称为函数的模块组成，从main()函数开始执行，函数由函数头和函数体组成，函数头指出函数返回值（如果有）类型和期望传递参数。函数体由一系列位于花括号{}中的C++语句组成。

C++语句包括以下6种

  • 声明语句：定义函数中使用的变量名称和类型
  • 赋值语句：使用赋值运算符=给变量赋值
  • 消息语句：将消息发送给对象，激发某种行为
  • 函数调用：执行函数，被调用的函数执行完毕后，程序返回到函数调用语句之后的语句
  • 函数原型：声明函数的返回类型，函数接受的参数数量和类型
  • 返回语句：将一个值从被调用函数哪里返回给调用函数中

类是用户定义的数据类型规范，描述了如何表示信息以及可对数据执行的操作。对象是根据类规范创建的实体。

C++提供了两个用于处理输入和输出的预定义对象（cin和cout），他们是istream和ostream类的实例，这两个类在iostream文件中定义。为ostream类定义的插入运算符<<将数据插入到输出流，为istream类定义的抽取运算符>>能从输入流抽取信息，cin和cout都是智能对象，能根据程序上下文自动将信息从一种形式转换为另一种形式。

什么是RPC

RPC可以分为两部分：用户调用接口 + 具体网络协议。
前者为开发者需要关心的，后者由框架来实现。

用户调用接口 

举个例子，我们定义一个函数，  
我们希望函数如果输入为“Hello World”的话，输出给一个“OK”，  
那么这个函数是个本地调用。  

如果一个远程服务收到“Hello World”可以给我们返回一个“OK”，  
那么这是一个远程调用。  
我们会和服务约定好远程调用的函数名。  
因此，我们的用户接口就是：输入、输出、远程函数名，  

比如用 SRPC 开发的话，client端的代码会长这样  
int main()
{
    Example::SRPCClient client(IP, PORT);
    EchoRequest req; // 用户自定义的请求结构
    EchoResponse resp; // 用户自定义的回复结构

    req.set_message("Hello World");
    client.Echo(&req, &resp, NULL); // 调用远程函数名为Echo
    return 0;
}

具体网络协议

具体网络协议，是框架来实现的，把开发者要发出和接收的内容以某种应用层协议打包进行网络收发。 

这里可以和HTTP进行一个明显的对比  
HTTP也是一种网络协议，但包的内容是固定的，必须是：请求行 + 请求头 + 请求体；  
RPC是一种自定义网络协议，由具体框架来定，比如SRPC里支持的RPC协议有：SRPC/thrift/BRPC/tRPC  

这些RPC协议都和HTTP平行，是应用层协议。我们再进一步思考，HTTP只包含具体网络协议，也可以返回比如我们常见的HTTP/1.1 200 OK，但仿佛没有用户调用接口，这是为什么呢？  
这里需要搞清楚，用户接口的功能是什么？最重要的功能有两个：  
1. 定位要调用的服务；  
2. 让我们的消息向前/向后兼容
定位要调用的服务消息前后兼容
http网络协议URL开发者自行在消息体里解决
rpc网络协议指定Service和Method名交给具体IDL

RPC和HTTP之间大部分都是协议互通的！是不是很神奇？接下来我们看一下RPC的层次架构，就可以明白为什么不同RPC框架之间、以及RPC和HTTP协议是如何做到互通的。
可以从SRPC的架构层次上来看，RPC框架有哪些层:

  • 用户代码（client的发送函数/server的函数实现）
  • IDL序列化（protobuf/thrift serialization）
  • 数据组织 （protobuf/thrift/json）
  • 压缩（none/gzip/zlib/snappy/lz4）
  • 协议 （Sogou-std/Baidu-std/Thrift-framed/TRPC）
  • 通信 （TCP/HTTP）
